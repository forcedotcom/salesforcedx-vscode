# W-22804372 — enable unicorn/no-single-promise-in-promise-methods

## Context

- enable `unicorn/no-single-promise-in-promise-methods: error` in `eslint.config.mjs`
- catches `Promise.all([p])` / `Promise.race([p])` / `Promise.any([p])` / `Promise.allSettled([p])` with single arg
- unicorn rules block in `eslint.config.mjs` (~line 178); insert alphabetically near other `no-*` rules
- ref PR: #7377

## Scope discipline

- ONLY changes required to land the lint rule: enable rule + mechanical unwrap of single-promise `Promise.all`/`race`/`any`/`allSettled` calls
- NO `Effect.gen` → `Effect.fn` refactors (rule does not require it)
- NO `Error` → `Schema.TaggedError` migrations (rule does not require it)
- if a violation site lives inside an `Effect.gen` / class-Error block, fix ONLY the unwrap; leave surrounding Effect idioms untouched

## Phases

### Phase 1 — enable rule + fix violations

- add `'unicorn/no-single-promise-in-promise-methods': 'error'` to unicorn block in `eslint.config.mjs` (alphabetical placement near other `no-*` rules)
- build local rules if needed: `npm run compile`
- enumerate violations: `npm run lint`
- fix patterns (mechanical):
  - `await Promise.all([p])` → `await p`
  - `const [x] = await Promise.all([p])` → `const x = await p`
  - `Promise.race([p])` / `Promise.any([p])` / `Promise.allSettled([p])` → `p` (or `await p`) preserving the original await/return semantics
  - empty `Promise.all([])` (if any surface): remove the line — it resolves to `[]` synchronously and is a no-op (note: rule targets single-element, not empty; empty arrays are out of scope unless flagged)
  - intentional cases (tuple return type, microtask boundary) → inline `// eslint-disable-next-line unicorn/no-single-promise-in-promise-methods` + reason
- include `scripts/**/*.ts` if covered by lint config
- commit message: `chore: enable unicorn/no-single-promise-in-promise-methods`
