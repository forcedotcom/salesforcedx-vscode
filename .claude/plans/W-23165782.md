- Export all tagged errors from `codeCoverageService.ts` (consult ts4023-effect-errors skill for the export-surface requirement that avoids TS4023 on the service's inferred error channel).

#### 1b — Effectful loops (no imperative `for`, no silent `catch {}`)

2 imperative `for` loops → `Effect.forEach`; silent `catch {}` → explicit `Effect.catchAll(() => Effect.void)`:

- L81-91 stat loop → `Effect.forEach(resultNames, name => statRecent(name), { concurrency: 'unbounded' })` collecting recent entries. Per-file `stat` failure → `Effect.catchAll(() => Effect.succeed(Option.none()))` + comment "skip files we can't stat". (Stat order does not matter; sort happens after.)
- L112-128 read loop → `Effect.forEach(filesToRead, name => readResult(name), { concurrency: 1 })`. **Must be `concurrency: 1` (sequential):** last-write-wins aggregation into `coverageByName` depends on chronological order (`sortByMtimeAscending`); unbounded concurrency would race the `Map.set` ordering. Per-file read/parse failure → `Effect.catchAll(() => Effect.void)` + comment "skip files we can't read or parse".
- `workspace.fs.*` calls wrap in `Effect.tryPromise` mapping to the appropriate tagged error (top-level readDirectory failure → `NoCoverageOnProjectError`).

Let `SettingsError` surface (rare; no catch).

### Phase 2 — Thin handler glue + runtime boundary + disposal

- Rewrite `colorizer.ts` `CodeCoverageHandler`: keep as Disposable glue only.

  - callback + `toggleCoverage`: `getApexTestingRuntime().runPromise(CodeCoverageService...)`, then apply decorations to editor from returned ranges.
  - No coverage state/logic in class; ranges come from service result.
- **Fix the subscription leak (`adversary:high`).** Today `window.onDidChangeActiveTextEditor(...)` (colorizer.ts:174) returns a Disposable that is silently dropped and never added to `Disposable.from(...)` at index.ts:211.
  - (a) Store the returned Disposable on the handler (`private readonly editorChangeSub: vscode.Disposable`).
  - (b) Implement `dispose(): void` on `CodeCoverageHandler` that calls `this.editorChangeSub.dispose()` — implement `vscode.Disposable` so it can join `Disposable.from`.
  - (c) Add `colorizer` to the `vscode.Disposable.from(...)` list at index.ts:211 (alongside `apexToggleColorizerCmd`, `statusBarToggle`).
- `handleCoverageException` (now in/called from service) catches via `catchTags` over the itemized errors (NoCoverageOnProjectError, StaleResultsError, NoCoverageForFileError, OutOfSyncCoverageError) — not `catchAll`. Each maps to channel-append (when disable-warnings) or `showWarningMessage`. Shared handling per tag is fine, but the catch must enumerate tags so a new failure type forces a compile decision.
- AC check: only runPromise = editor-change/toggle callback boundary.
- commit: `refactor(apex-testing): reduce CodeCoverageHandler to disposable vscode edge glue - W-23165782`
