# W-23069601 — Migrate sf.org.open to services (POC)

## Context

- POC gating all 4.x service migrations. Land + review first; fold learnings into 4.x.
- Simplest CLI command chosen as proof: `org open --url-only --json` → `openExternal`.
- Current: `packages/salesforcedx-vscode-org/src/commands/orgOpen.ts` — `SfCommandlet` + 2 `SfCommandletExecutor`s:
  - `OrgOpenExecutor` (non-container): runs `sf org:open` via CLI, CLI opens browser.
  - `OrgOpenContainerExecutor` (container): `org:open --url-only --json` via `CliCommandExecutor`, parses stdout w/ `OrgOpenContainerResultParser`, `vscode.env.openExternal`, appends msg to channel.
- Model to copy: `orgDeleteDefaultCommand` (same file's sibling `orgDelete.ts`) — `Effect.fn` cmd, `api.services.TerminalService.simpleExec`, `api.services.ChannelService.appendToChannel`, registered via `registerCommandWithLayer(AllServicesLayer)` in `index.ts`.
- Services available: `TerminalService.simpleExec({command, parse, timeout, env?})` (desktop-only, fails web w/ `TerminalServiceError`; auto-injects `SF_JSON_TO_STDOUT=true` + `FORCE_COLOR=0` for `sf ` commands, caller `env` wins); `ChannelService.appendToChannel`; `ProjectService.getSfProject()`; `TargetOrgRef` (SubscriptionRef w/ `username`).
- Constraints (all load-bearing):
  - clean JSON stdout: `TerminalService.simpleExec` auto-injects `SF_JSON_TO_STDOUT=true` + `FORCE_COLOR=0` for `sf ` commands; callers no longer pass these flags.
  - default-org resolution: `simpleExec` runs without workspace cwd; must pass `--target-org <username>` from `TargetOrgRef` (orgDelete pattern).
  - project precondition: gated on `ProjectService.getSfProject()`, which sets `sf:project_opened` context and fails with `FailedToResolveSfProjectError` (typed error, rendered by ErrorHandlerService) when no SF project.
- Registration: `index.ts` `activateEffect` already does `registerCommand('sf.org.delete.default', orgDeleteDefaultCommand)`; `ORG_OPEN_COMMAND = 'sf.org.open'` currently registered via plain `vscode.commands.registerCommand(ORG_OPEN_COMMAND, orgOpen)`.
- Channel name = displayName `Salesforce Org Management`.
- `openExternal`: no service wrapper; call `vscode.env.openExternal(URI.parse(url))` inside `Effect.sync`/`Effect.promise` (orgDelete calls `vscode` directly via `Effect.sync`).

## Done when

- `sf.org.open` runs as `Effect.fn` command registered via `registerCommandWithLayer`.
- Uses `TerminalService.simpleExec` for `org open --url-only --json`, opens url via `openExternal`, appends access message to channel.
- Old `SfCommandlet`/executor path + `OrgOpenContainerResultParser` removed (or parser retained only if still referenced).
- Jest + new e2e green.

## Phases

### Phase 0 — extend TerminalService for env auto-injection

- `sf org open --json` parsing depends on `SF_JSON_TO_STDOUT=true` + `FORCE_COLOR=0`. Resolve at the service seam:
  - `childProcess.ts`: add `env?: Record<string, string>` to `ExecOptions` type; merge into node exec call — spread `process.env` first, else node exec drops parent env if partial env passed.
  - `terminalService.ts` `simpleExec`: add optional `env?` param; for `sf ` commands, auto-inject `{ SF_JSON_TO_STDOUT: 'true', FORCE_COLOR: '0' }` (merged under caller `env` so explicit overrides still win). Thread into `childProcess.exec`. Annotate span w/ env key names (not values — no secrets).
  - jest (services pkg): assert `simpleExec` forwards merged env (process.env + auto-injected + caller override) to the injected `ChildProcess.exec` mock.
- commit: `feat(services): TerminalService.simpleExec auto-injects env for sf commands - W-23069601`

### Phase 1 — orgOpenCommand Effect + registration

- New `orgOpenCommand = Effect.fn('orgOpenCommand')` in `orgOpen.ts`:
  - get api.
  - **precondition (load-bearing)**: `yield* api.services.ProjectService.getSfProject()`. Sets `sf:project_opened` context; fails w/ `FailedToResolveSfProjectError` (typed, rendered by ErrorHandlerService) if no project. Bails without running CLI.
  - **target-org (avoids cwd dependency)**: `const orgInfo = yield* SubscriptionRef.get(yield* api.services.TargetOrgRef())`; build `const targetOrgFlag = orgInfo.username ? \` --target-org ${orgInfo.username}\` : ''` (same as orgDelete.ts:172). If no username, the command still runs (sf resolves default) but log/annotate the gap.
  - **env (clean JSON)**: `terminalService.simpleExec({ command: \`sf org open --url-only --json${targetOrgFlag}\`, parse })` — simpleExec auto-injects `SF_JSON_TO_STDOUT=true` + `FORCE_COLOR=0` for `sf ` commands.
  - `parse` → **tagged** discriminated union (NOT a structural `'result' in x` check):
    - `OrgOpenSuccess = Schema.Struct({ _tag: Schema.Literal('OrgOpenSuccess'), result: Schema.Struct({ url, orgId, username }) })`.
    - `OrgOpenFailure = Schema.Struct({ _tag: Schema.Literal('OrgOpenFailure'), status: Schema.Literal(1), message: Schema.String })` (preserves old `OrgOpenErrorResult` channel-message behavior, orgOpen.ts:83-88).
    - `OrgOpenResponse = Schema.Union(OrgOpenSuccess, OrgOpenFailure)`.
    - sf CLI emits `{ result }` (success) / `{ status, message }` (failure) — neither carries `_tag`. So `parse` must JSON.parse then **inject `_tag`** before decode: `const raw = JSON.parse(stdout); const tagged = { ...raw, _tag: 'result' in raw ? 'OrgOpenSuccess' : 'OrgOpenFailure' }`, then `Schema.decodeUnknown(OrgOpenResponse)(tagged)`. (`'result' in raw` only in the parse adapter; all dispatch via `_tag`).
    - Schema decode (not raw assert) so malformed stdout → tagged decode error, not bare `SyntaxError`.
  - dispatch on the decoded response with `Match` (per effect-best-practices "if/else or switch on tagged union → `Match.type<T>().pipe(Match.tag(...), Match.exhaustive)`"):
    ```
    yield* Match.type<OrgOpenResponse>().pipe(
      Match.tag('OrgOpenSuccess', ({ result }) => Effect.gen(function* () {
        yield* Effect.sync(() => void vscode.env.openExternal(URI.parse(result.url)));
        yield* channel.appendToChannel(<org_open_container_mode_message_text w/ orgId/username/url>);
        const outputChannel = yield* channel.getChannel;
        yield* Effect.sync(() => outputChannel.show());
      })),
      Match.tag('OrgOpenFailure', ({ message }) => channel.appendToChannel(message)),
      Match.exhaustive
    )(response);
    ```
    where `const channel = yield* api.services.ChannelService` is already in scope.
  - **channel show**: NO legacy singleton. Do not `import { channelService } from '../channels'`. Use the in-layer service: `const outputChannel = yield* channel.getChannel` then `yield* Effect.sync(() => outputChannel.show())` (pattern: executeAnonymousService.ts:208 `yield* Effect.sync(() => channel.show())`, sourceTrackingService.ts:381; channelService.ts:20 `getChannel: Effect.sync(() => channel)`).
- `index.ts`: drop `vscode.commands.registerCommand(ORG_OPEN_COMMAND, orgOpen)` from `registerCommands()`; add `yield* registerCommand(ORG_OPEN_COMMAND, orgOpenCommand)` in `activateEffect` next to `sf.org.delete.default`.
- Remove old `OrgOpenExecutor`, `OrgOpenContainerExecutor`, `getExecutor`, `orgOpen`, `isSFContainerMode`, `OrgOpenContainerResultParser`. Delete `parsers/orgOpenContainerResultParser.ts` + export if unused.
- Drop unused imports (`CliCommandExecutor`, `SfCommandlet`, etc.). **Explicitly drop `import { channelService } from '../channels'`** (legacy singleton) — use in-layer `ChannelService.getChannel`. Add `SubscriptionRef`, `Schema`, `Match`, `nls` keys.
- commit: `refactor(org): migrate sf.org.open to services via TerminalService - W-23069601`
