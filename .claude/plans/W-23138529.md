# W-23138529: ESLint rule `local/no-successive-annotate-current-span`

## Context

- ESLint rule flags back-to-back `Effect.annotateCurrentSpan`, autofix → single merged call.
- `packages/eslint-local-rules` (`@salesforce/eslint-plugin-vscode-extensions`). Rule src per-file in `src/`, registered in `src/index.ts`, tested via `@typescript-eslint/rule-tester` in `test/`.
- Convention ref: `src/requireEffectFnSpanName.ts` (RuleCreator.withoutDocs, CallExpression visitor), `src/noVscodeUri.ts`. Fixer + test-output ref: `src/noEffectFnWrapper.ts`, `test/no-effect-fn-wrapper.test.ts`.
- Enabled in Effect block: `eslint.config.mjs` ~629-695 (same `files` glob as `effect/no-import-from-barrel-package`).
- Real successive usage (verified via `rg "annotateCurrentSpan"` over all non-test `*.ts`, 2026-06-23): ONLY two forms exist in the wild:
  - generator (consecutive `yield*`): `metadata/src/commands/generateManifest.ts:52-53` (2-arg `(key,value)`).
  - `Effect.tap` chain: `packageInstall.ts:178-184` (3 adjacent `.tap(x => Effect.annotateCurrentSpan('k', v))`).
- NOT successive (valid-test cases, confirmed in tree): `projectInit.ts:45,47` (generator, intervening `const dirsToCreate` between the two annotates); `connectionService.ts:371` (`Effect.all`-style array `[annotateCurrentSpan(...), SubscriptionRef.set(...)]` — second element is a different effect, not an annotate).
- NO `Effect.all`/`forEach` array nor `andThen` chain anywhere contains successive `annotateCurrentSpan` elements. Single-arg object form `({...})` exists abundantly standalone but never back-to-back.

### Scope (strictly successive only)

- ≥2 adjacent `Effect.annotateCurrentSpan`, no intervening statements/elements.
- Signatures merge → 1 object call:
  - `(key, value)` → `key: value` (string-literal → ident/quoted)
  - `({...})` → spread props
  - `(keyVar, val)` → `[keyVar]: val` (computed)
- Forms (scoped to verified real usage): **generator (consecutive `yield*`)** and **`Effect.tap` chain (adjacent `.tap` w/ annotate-only arrow)** ONLY. Array (`Effect.all`/`forEach`) and `andThen` detection are explicitly OUT OF SCOPE — no real usage; adding speculative AST traversal risks false positives. Add later if usage surfaces.
- Duplicate keys (same merged key appears ≥2×): report `duplicateKey` error. Resolution is part of the single autofix (keep-last); NOT a separate ESLint suggestion.
- Autofix: replace run w/ single `Effect.annotateCurrentSpan({ ...merged })`.

NO ESLint `suggest`/`hasSuggestions` — no existing rule in `eslint-local-rules` uses suggestions (verified: only matches are unrelated string literals). Duplicate keys are resolved inside the single autofix, not via a separate suggestion. `meta.hasSuggestions` is OMITTED.
  - `isAnnotateCurrentSpanCall` (Effect member check).
  - Arg→prop extraction: 1-arg ObjectExpression → props; 2-arg literal key → prop; 2-arg non-literal key → computed `[k]`.
