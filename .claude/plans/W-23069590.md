# W-23069590 — 1.2 Add ConnectionService.listAllAuthorizations

## Context

- Wrap `AuthInfo.listAllAuthorizations()` (core static, `Promise<OrgAuthorization[]>`) in `Effect.fn` w/ `Schema.TaggedError`.
- Centralizes core dep; consumers (`vscode-org` orgUtil, parameterGatherers, orgList) currently call core directly w/ ad-hoc `catch: e => e`. Not migrating consumers here — 1.2 just adds the method (series WI).
- File: `packages/salesforcedx-vscode-services/src/core/connectionService.ts`.
- `ConnectionService` = `Effect.Service`; existing methods `getConnection`, `invalidateCachedConnections` returned from `effect` gen. Existing errors follow `Schema.TaggedError` pattern w/ `message` + optional `cause: Schema.instanceOf(Error)`. `unknownToErrorCause` from `./shared`.
- **Public API: yes.** `listAllAuthorizations` is intended for external consumers (vscode-org). `ConnectionService` is exported as a type (`export { type ConnectionService }`, index.ts ~439) and consumed via `api.services.ConnectionService.getConnection()` (orgUtil.ts:111-112). The exported type captures the `effect` gen return object, so adding `listAllAuthorizations` to that returned object automatically surfaces it on the public API — **no separate per-method export line is needed** (mirrors how `getConnection`/`invalidateCachedConnections` are surfaced). Only the new *error* type needs an explicit `export type` in index.ts (TS4023).
- Series WI migrates the existing ad-hoc consumers to this method: orgUtil.ts (lines 47, 264, 444), parameterGatherers (selectOrgForDisplay/selectOrgsForLogout/selectDeletableOrg), orgList.ts:190. 1.2 only adds the method.
- DoD: new exported `FailedToListAuthorizationsError`-tagged error + `listAllAuthorizations` method on `ConnectionService` (public, reachable via `api.services.ConnectionService`); error exported from `index.ts`; method present on exported `ConnectionService` type surface (verified by `npm run compile` of a consumer ref / type check, no TS4023); unit test covers success + error.

- export the new **error** type in `index.ts` — append `FailedToListAuthorizationsError` to the existing `export type { ... } from './core/connectionService'` block (lines 178-184) — req'd to avoid TS4023. No change needed to the `export { type ConnectionService }` line (line ~439) — the method is surfaced automatically through the service type.
