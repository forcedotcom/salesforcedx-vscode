# W-22691592 — Apex `in-workspace` filter hides LWC tests

GitHub issue: https://github.com/forcedotcom/salesforcedx-vscode/issues/7350

## Bug

Typing the Test Explorer tag filter `@sf.apex.testController:in-workspace` (documented as `@in-workspace`)
to narrow Apex tests to those in the current workspace also makes the **entire LWC tests section disappear**.

## Root cause

VS Code's Test Explorer tag filter matches on the `TestTag.id` string and filters the **whole** test tree
across **all** controllers — any item lacking a tag with that id is hidden.

- Apex controller tags workspace items with `new vscode.TestTag('in-workspace')`
  - [testController.ts:94](../../packages/salesforcedx-vscode-apex-testing/src/views/testController.ts#L94)
  - applied in [orgTestItems.ts:205-206](../../packages/salesforcedx-vscode-apex-testing/src/views/orgTestItems.ts#L205-L206) and [testController.ts:818-831](../../packages/salesforcedx-vscode-apex-testing/src/views/testController.ts#L818-L831)
- LWC controller assigns **no tags at all** to its file/case items
  - [lwcTestController.ts:213](../../packages/salesforcedx-vscode-lwc/src/testSupport/testExplorer/lwcTestController.ts#L213) (file items) and [:232](../../packages/salesforcedx-vscode-lwc/src/testSupport/testExplorer/lwcTestController.ts#L232) (case items)

So with the `in-workspace` filter active, every untagged LWC item is filtered out.

All LWC Jest tests are discovered from the workspace, so they are inherently "in-workspace" and should
survive the same filter.

## Fix

Tag LWC test items with an `in-workspace` `TestTag` (matching id `'in-workspace'`) so they remain visible
when the shared `@in-workspace` filter is applied.

### Changes — [lwcTestController.ts](../../packages/salesforcedx-vscode-lwc/src/testSupport/testExplorer/lwcTestController.ts)

1. Add a private field and construct the tag in the constructor:
   ```ts
   private readonly inWorkspaceTag = new vscode.TestTag('in-workspace');
   ```
   (field initializer; no constructor edit needed beyond field declaration)
2. In `getOrCreateFileItem` (~line 213), after `createTestItem`, set `item.tags = [this.inWorkspaceTag];`
3. In `resolveFileChildren` (~line 232), when creating a new `caseItem`, set `caseItem.tags = [this.inWorkspaceTag];`

Notes:
- Do **not** add a run profile tagged with `in-workspace` — that would change which profiles the toolbar
  Run button offers for LWC. Only item `tags` drive the visibility filter; leave LWC profiles as-is.
- Tag id must be exactly `'in-workspace'` to match the same filter Apex uses.

## Out of scope / open question

- Should LWC also gain an `org-only`/`stale` notion? No — LWC Jest tests have no org/stale concept; only the
  workspace-visibility filter must not hide them. Single `in-workspace` tag is sufficient.
