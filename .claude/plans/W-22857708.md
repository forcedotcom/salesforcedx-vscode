# W-22857708 ‚Äî local/no-self-barrel-import

## Context

- ban bare-ancestor barrel imports: `'.'`, `'..'`, `'../..'`, `'../../..'`, etc.
- allow named imports like `'./foo'`, `'../foo'`, external pkgs
- pattern follows `noDirectServicesImports` (`RuleCreator.withoutDocs`)
- known violations:
  - `packages/salesforcedx-vscode-core/src/context/workspaceContext.ts` ‚Äî `from '.'`
  - `packages/salesforcedx-utils-vscode/src/context/workspaceContextUtil.ts` ‚Äî `from '..'`
  - `packages/salesforcedx-utils-vscode/src/telemetry/reporters/telemetryFile.ts` ‚Äî `from '../..'`
  - `packages/salesforcedx-utils-vscode/src/helpers/paths.ts` ‚Äî `from '..'`
  - `packages/salesforcedx-visualforce-markup-language-server/src/services/htmlLinks.ts` ‚Äî `from '..'` (note: package src ignored by eslint config; still fix for consistency)

## Phases

### 1. Add rule + tests

- new `packages/eslint-local-rules/src/noSelfBarrelImport.ts`
  - regex: `^\.(\.\/\.\.)*$|^\.\.(\/\.\.)*$` ‚Äî matches `.`, `..`, `../..`, `../../..`
  - fail on `ImportDeclaration` whose source string matches
  - no autofix (correct target depends on named imports used)
  - single `messageId: noSelfBarrel`: ``Importing from `{{source}}` resolves to the package's barrel (`{{source}}/index.ts`). Replace with direct import from file defining each export (open barrel to find source).``
  - placeholder `{{source}}` filled from `node.source.value`
- register `'no-self-barrel-import'` in `packages/eslint-local-rules/src/index.ts`
- new `packages/eslint-local-rules/test/no-self-barrel-import.test.ts`
  - valid: `from './foo'`, `from '../foo'`, `from 'pkg'`, `from '../../foo/bar'`
  - invalid: `from '.'`, `from '..'`, `from '../..'`, `from '../../..'`
  - assert message contains the offending source string (verifies placeholder rendering)
- commit: `feat: add local/no-self-barrel-import rule`

### 2. Wire into eslint config

- add `'local/no-self-barrel-import': 'error'` in `eslint.config.mjs` under `**/*.ts` block (alongside other `local/*` rules)
- commit: `chore(eslint): enable no-self-barrel-import repo-wide`

### 3. Fix existing violations

- for each violation: open barrel, locate export source, replace with direct file imports (split when imports span files)
  - `core/context/workspaceContext.ts` (`from '.'` ‚Üí `context/index.ts`): replace with `from './workspaceContextUtils'` (or specific files per named import)
  - `utils-vscode/context/workspaceContextUtil.ts` (`from '..'` ‚Üí `src/index.ts`): split into direct imports (e.g., `from '../providers/configAggregatorProvider'` + `from '../services/telemetry'`) per actual named imports
  - `utils-vscode/telemetry/reporters/telemetryFile.ts` (`from '../..'` ‚Üí `src/index.ts`): replace with `from '../../helpers/paths'` or specific file(s)
  - `utils-vscode/helpers/paths.ts` (`from '..'` ‚Üí `src/index.ts`): replace with `from '../context/workspaceContextUtil'` or specific file(s)
  - `visualforce-markup-language-server/services/htmlLinks.ts` (`from '..'`): replace with the specific re-exporting file
- commit: `refactor: replace bare-ancestor barrel imports`
