# W-22857708 — local/no-self-barrel-import

## Context

- ban bare-ancestor barrel imports: `'.'`, `'..'`, `'../..'`, `'../../..'`, etc.
- allow named imports like `'./foo'`, `'../foo'`, external pkgs
- pattern follows `noDirectServicesImports` (`RuleCreator.withoutDocs`)
- known violations:
  - `packages/salesforcedx-vscode-core/src/context/workspaceContext.ts` — `from '.'`
  - `packages/salesforcedx-utils-vscode/src/context/workspaceContextUtil.ts` — `from '..'`
  - `packages/salesforcedx-utils-vscode/src/telemetry/reporters/telemetryFile.ts` — `from '../..'`
  - `packages/salesforcedx-utils-vscode/src/helpers/paths.ts` — `from '..'`
  - `packages/salesforcedx-visualforce-markup-language-server/src/services/htmlLinks.ts` — `from '..'` (note: package src ignored by eslint config; still fix for consistency)

## Phases

### 1. Add rule + tests

- new `packages/eslint-local-rules/src/noSelfBarrelImport.ts`
  - regex: `^\.(\.\/\.\.)*$|^\.\.(\/\.\.)*$` — matches `.`, `..`, `../..`, `../../..`
  - fail on `ImportDeclaration` whose source string matches
  - no autofix (barrel may re-export many files; correct target depends on which named imports are used)
  - single `messageId: noSelfBarrel` with message that names the resolved barrel and tells the dev to inspect it: ``Importing from `{{source}}` resolves to the package's barrel (`{{source}}/index.ts`) which re-exports multiple files. Replace with a direct import from the specific file that defines each named export (open the barrel to find the source).``
  - placeholder `{{source}}` filled from `node.source.value`
- register `'no-self-barrel-import'` in `packages/eslint-local-rules/src/index.ts`
- new `packages/eslint-local-rules/test/no-self-barrel-import.test.ts`
  - valid: `from './foo'`, `from '../foo'`, `from 'pkg'`, `from '../../foo/bar'`
  - invalid: `from '.'`, `from '..'`, `from '../..'`, `from '../../..'`
  - assert message contains the offending source string (verifies placeholder rendering)
- commit: `feat: add local/no-self-barrel-import rule`

### 2. Wire into eslint config

- add `'local/no-self-barrel-import': 'error'` in `eslint.config.mjs` under `**/*.ts` block (alongside other `local/*` rules)
- commit: `chore(eslint): enable no-self-barrel-import repo-wide`

### 3. Fix existing violations

- procedure for each violation: open the resolved `index.ts` barrel, locate which file each named import is re-exported from, replace the bare-ancestor specifier with one or more direct file imports (split into multiple `import` statements when imports come from different files)
  - `core/context/workspaceContext.ts` (`from '.'` → `context/index.ts`): replace with `from './workspaceContextUtils'` (or specific files per named import)
  - `utils-vscode/context/workspaceContextUtil.ts` (`from '..'` → `src/index.ts`): split into direct imports (e.g., `from '../providers/configAggregatorProvider'` + `from '../services/telemetry'`) per actual named imports
  - `utils-vscode/telemetry/reporters/telemetryFile.ts` (`from '../..'` → `src/index.ts`): replace with `from '../../helpers/paths'` or specific file(s)
  - `utils-vscode/helpers/paths.ts` (`from '..'` → `src/index.ts`): replace with `from '../context/workspaceContextUtil'` or specific file(s)
  - `visualforce-markup-language-server/services/htmlLinks.ts` (`from '..'`): replace with the specific re-exporting file
- commit: `refactor: replace bare-ancestor barrel imports`
