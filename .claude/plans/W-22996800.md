  4. Effect error channels (Schema.TaggedError); no `throw`/`try-catch` in Effect. 1 tagged error/failure mode; catch only to ignore/improve msg. Defensive try-catch breaks telemetry, swallows failures. Src: effect-best-practices, ts4023-effect-errors skills.

  7. Reuse `api.services.prebuiltServicesDependencies`; never `.Default` to rebuild a prebuilt service. Rebuild = 2nd singleton (dup watchers/caches/org refs), still compiles. Most common LLM error per skill. Src: services-extension-consumption skill.

Do NOT name "org-browser" generically; cite the specific package and its verified dep state. Src: CodeReuse.md:35-39, services README, Extensions.md:66-68, verified package.json states.
  9. Reduce CLI deps; web extensions MUST avoid CLI entirely. General preference = fewer sf/sfdx shell-outs (use in-process libs/services). Hard requirement for web (no CLI in browser). BUT known-never-web features (e.g. org ext) may use CLI — don't force CLI-removal on desktop-only code. Src: web-support mechanism, `terminalService.ts` web guard.
