### Phase 2: implement apexTestSuiteRemove in apexTestSuite.ts

- Define a local type `RemovableClassItem` extending `ApexTestQuickPickItem` with an additional `membershipId: string` property to carry the TestSuiteMembership record ID without polluting the user-visible `description` field.
- Add `gatherRemoveOptions` Effect.fn:
  - Pick suite via `listApexTestSuiteItems`
  - Escape the suiteId with `suiteId.replaceAll("'", "''")` before embedding in SOQL (consistent with codebase escaping pattern in testController.ts and orgApexClassProvider.ts)
  - Query `SELECT Id, ApexClassId FROM TestSuiteMembership WHERE ApexTestSuiteId = '<escapedSuiteId>'` via `connection.tooling.query`
  - Resolve class names: query `SELECT Id, Name, NamespacePrefix FROM ApexClass WHERE Id IN (...)` via tooling (also escape class IDs with `replaceAll("'", "''")`)
  - Show multi-select quick pick using `RemovableClassItem[]` with `label=Name`, `description=namespacePrefix ?? ''` (human-readable, consistent with `listApexClassItems`), and `membershipId` set to the TestSuiteMembership record Id
  - Return selected `membershipId` values from the picked items
- Add `removeSuiteMembers` Effect.fn:
  - Delete selected membership records via `connection.tooling.delete('TestSuiteMembership', ids)`
  - Append `Ended` sentinel to channel
  - Show success notification
  - `clearAllSuiteChildren` + `refresh` on test controller
- Export `apexTestSuiteRemove` Effect.fn wiring gather + remove
