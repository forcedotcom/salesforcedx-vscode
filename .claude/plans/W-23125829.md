# W-23125829: gen→fn services core (6 sites)

## Context

Convert arrow-wrapper `=> Effect.gen` to `Effect.fn("name")` per effect-best-practices (`Effect.fn` > `Effect.gen`; gen only for shared pipes). Pull params into generator, no closure capture. All in `salesforcedx-vscode-services/src/core`.

`Effect.fn` w/ params returns a function; zero-param `=> Effect.gen` becomes `Effect.fn("name")(function* () {...})`, still called as `fn()` — matches existing `clearDefaultOrgRef`. No caller changes.

Trailing `.pipe(Effect.withSpan(...))` redundant once `Effect.fn` named span present. Rule: **span name moves verbatim to the `Effect.fn` name**; any `attributes` move to `Effect.annotateCurrentSpan({...})` as the first `yield*` inside the generator. This keeps span name + attributes byte-identical (telemetry parity).
