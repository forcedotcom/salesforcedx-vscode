- Add `preinstall require-root-install.js`

### Phase 4 — Convert 31 test files from chai/sinon/mocha to jest
NOT a config-only swap. `mocha-compat` does not exist for jest; no chai/sinon anywhere in monorepo (`grep -rn '"sinon"\|"chai"' packages/*/package.json` → none); no jest-sinon bridge. Each of the 31 `test/**` files needs a real rewrite:
- Assertions: chai `expect(...).to.equal/deep.equal/be.true/throw` + `assert(...)` → jest `expect(...).toBe/toEqual/.toBe(true)/.toThrow` (e.g. `test/execute/executeService.test.ts:13 import { assert, expect } from 'chai'`)
- Stubs/spies: sinon `createSandbox()/sandbox.stub()/sandbox.spy()/.restore()` → `jest.spyOn`/`jest.mock`/`jest.fn`, `afterEach(jest.restoreAllMocks)` (e.g. `test/tests/testService.test.ts:11 import { createSandbox, SinonSandbox, SinonStub, spy } from 'sinon'`)
- KEEP `@salesforce/core` `TestContext` — jest-compatible, already used in monorepo jest suites (e.g. `packages/salesforcedx-vscode-apex/test/jest/languageServerOrphanHandler.test.ts`)
- Hooks: mocha `before/after` → jest `beforeAll/afterAll`; `describe/it` carry over
- Add `jest.config.js` extending `config/jest.base.config.js` (ts-jest preset, `setupFilesAfterEnv` per base); add `test/tsconfig.json` if pattern requires (see `salesforcedx-utils`)
- Drop devDeps: `mocha`, `chai`, `sinon`, `@types/*` for those, `nyc`
- SIZING: ~31 files, mechanical but per-file. Largest phase. Budget accordingly; do not batch blindly — sinon stub semantics (call-args matchers, `resolves`/`rejects`) need careful 1:1 mapping.
- Goal: `npm run test -w packages/salesforcedx-apex` green.
- Files: `packages/salesforcedx-apex/{jest.config.js,test/tsconfig.json,test/**,package.json}`
- Commit: `test(apex): convert chai/sinon/mocha suite to jest - W-23051181`
