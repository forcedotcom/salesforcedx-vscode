# W-21973088 — Refactor HashableUri: wrap URI instead of extending

## Context

- `HashableUri` extends `vscode-uri` `URI`. Cross-bundle `instanceof URI` fails (each ext bundles own copy) → `[Equal.symbol]` broken across bundles. `diffTypes.ts:13-19` uses structural duck-type as workaround.
- Inheritance forces `.toUri()` strip-back at every vscode RPC site.
- Target: type + namespace `{ readonly uri: URI }` + Hash/Equal symbols. Structural Equal. API = `fromUri` + `with` only.
- Atomic refactor; lint already blocks deep prod imports. Only 4 test files use deep import.
- Adversary findings folded in below: every direct property/Util/uriToPath call against a HashableUri-typed value must be migrated to `.uri`.

## Phases

### Phase 1 — Rewrite hashableUri.ts + value re-export

- `packages/salesforcedx-vscode-services/src/vscode/hashableUri.ts`: replace `class extends URI` with `type HashableUri = { readonly uri: URI; readonly [Hash.symbol]: () => number; readonly [Equal.symbol]: (that: unknown) => boolean }` + namespace `HashableUri { fromUri, with }`.
- Drop `file/parse/from/toUri/protected ctor/instance with`. `fromUri` does Windows drive lowercase via `uri.with({ path })` when scheme === 'file' and path matches `/^\/[A-Z]:/`.
- Structural `isHashableUriShape(u): u is HashableUri` checks `'uri' in u && typeof u.uri === 'object' && u.uri !== undefined && typeof u.uri.scheme === 'string'`.
- Equal: structural — both shape-valid, `a.uri.toString() === b.uri.toString()`. Hash: `Hash.string(a.uri.toString())`.
- `with` dual-signature via `effect/Function` `dual`. Define helper type before namespace:
  ```ts
  type UriChange = Parameters<URI['with']>[0]; // { scheme?; authority?; path?; query?; fragment? }
  ```
  Then:
  ```ts
  with: dual(2, (hu: HashableUri, change: UriChange): HashableUri => fromUri(hu.uri.with(change)));
  ```
- `packages/salesforcedx-vscode-services/src/index.ts:166`: `export type` → `export` (value+type via namespace merge).
- Commit msg: `refactor(services): wrap URI in HashableUri instead of extending`
