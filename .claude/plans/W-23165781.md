# W-23165781 — apex-testing: drop telemetry shim + OUTPUT_CHANNEL proxy

## Context
2 activation-block cleanups, bundled so later WIs don't fight `index.ts`. Pkg: `salesforcedx-vscode-apex-testing`.

- Telemetry shim `src/telemetry/telemetry.ts` = backward-compat wrapper. Current shim is the fire-and-forget anti-pattern: `runPromise(...).catch(() => {})` (telemetry.ts:13-22) silently swallows errors — FORBIDDEN (effect SKILL.md anti-patterns: swallowing errors / `.catch(() => {})`). Do NOT replicate it. Spans already emitted via `registerCommandWithRuntime` + `Effect.withSpan`. 3 refs:
  - `index.ts:233` `sendExtensionDeactivationEvent()` — drop
  - `views/testController.ts:1356,1537` `sendEventData(...)` — inline to direct span (preserve `apexTestRun`/`apexTestReportGenerated` events) via `getApexTestingRuntime().runFork(...)` (NOT runPromise+catch). `runFork` surfaces failures to the runtime error channel instead of swallowing them; pattern: `services/src/vscode/registerCommand.ts:66` `runtime.runFork(f().pipe(Effect.withSpan(...)))`. `getApexTestingRuntime()` returns a `ManagedRuntime` exposing `.runFork`.
- OUTPUT_CHANNEL Proxy in `channels/index.ts` — sync handle for `.show()`. 3 callers, all Effect pipelines w/ `api`/`channelService` already in scope:
  - `commands/apexTestRun.ts:126` (channelService @100)
  - `commands/apexTestRunCodeAction.ts:82` (channelService @43)
  - `commands/apexTestSuite.ts:119` (channelService @106, inside `buildSuite`)
  - replace w/ `yield* channelService.showChannel` — `showChannel` is a first-class `Effect` member of `ChannelService` (`services/src/vscode/channelService.ts:22` `showChannel: Effect.sync(() => channel.show())`). Use it directly; do NOT use `(yield* channelService.getChannel).show()` (skips the idiomatic API, two yields where one suffices).
- Stale jest mocks to clean: `test/jest/views/testController.test.ts:79` (telemetry mock), `test/jest/utils/testReportGenerator.test.ts:19` (OUTPUT_CHANNEL + nonexistent `channelService` export; src no longer uses)
- BLOCKER: keep `Ended SFDX:` channel sentinel intact (gated by #7565). Sentinel = `appendEnded` (`appendToChannel`), untouched — only `.show()` changes.

### Phase 2 — telemetry shim removal
- `index.ts`: drop `sendExtensionDeactivationEvent()` call (233) + import (41)
- DELETE `src/telemetry/telemetry.ts` (`logTelemetry` shim) entirely — telemetry emitted via `Effect.annotateCurrentSpan` on the relevant span; event name = span name, duration automatic (OTel-first; AppInsights de-emphasized)
- `apexTestRunUtils.ts`: annotate enclosing `runApexTests` span — `yield* Effect.annotateCurrentSpan({ trigger, testsRan, testsPassed, testsFailed })`. Report telemetry: `outputFormat` annotated inside `writeAndOpenTestReport`
- `testController.ts` test-run site (`runTests`, plain async, no ambient span): wrap forked effect in span — `getApexTestingRuntime().runFork(Effect.annotateCurrentSpan({ trigger: 'testController', isDebug: String(isDebug), testsRan: testCount }).pipe(Effect.withSpan('apexTestRun')))`. Drop hand-computed `durationMs`/`startTime`
  `runFork` not `runPromise+catch` — errors surface to runtime error channel, not swallowed
- `testController.ts` report site: `getApexTestingRuntime().runFork(writeAndOpenTestReport(...).pipe(Effect.tap(() => Effect.annotateCurrentSpan({ trigger: 'testExplorer' })), Effect.withSpan('apexTestReportGenerated'), Effect.catchAllCause(cause => Effect.logError('Failed to generate test report', cause))))`. Drop `reportDurationMs`/`reportStartTime`
  net: drops `await`+`try/catch`+`console.error`; `catchAllCause`→`logError` preserves continue-on-failure
  trade-off (accepted): `trigger` on wrapper span, `outputFormat` on separately-forked `writeAndOpenTestReport` span → two spans not one merged event. Fine OTel-first
- `testController.test.ts`: drop telemetry jest.mock (79)
