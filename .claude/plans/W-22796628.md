### Phase 4 — CI workflow

**Decision: create new `.github/workflows/apexLspE2E.yml`** mirroring `apexTestingE2E.yml` structure (no `e2e-web` job — apex LSP is desktop-only; only `e2e-desktop` matrix). Rationale: existing apex-* Playwright workflows are one-per-package; apexE2E.yml is a WDIO-only file that calls `runE2ETest.yml` and is being depleted. Adding a job to a WDIO file would mix runners.

Concrete workflow shape:

- `name: Apex LSP E2E (Playwright)`
- triggers: `workflow_dispatch`, `workflow_call` (with `vscode_version` input), `push: branches-ignore: [main, develop]` + same `paths-ignore` as apexTestingE2E.yml
- `concurrency.group: ci-${{ github.ref }}-apexLspE2E`
- single job `e2e-desktop`:
  - matrix `os: [macos-latest, ubuntu-latest, windows-latest]` with `xvfb-run -a ` prefix on ubuntu (mirrors apexTestingE2E.yml:170-179)
  - same env: `VSCODE_DESKTOP=1`, `E2E_FROM_VSIX=1`, `SF_TEMP_SHOW_SECRETS=true`, `NON_TRACKING_ORG_ALIAS=nonTrackingTestOrg`, `PLAYWRIGHT_VSCODE_VERSION` resolution
  - same try-run / cache-miss → install CLI / auth dev hub / install Playwright / generate project / create scratch / parallel-run / recreate-org / sequential retry / span copy / artifact upload / cleanup pattern (copy verbatim from apexTestingE2E.yml steps, swap `salesforcedx-vscode-apex-testing` → `salesforcedx-vscode-apex` in workspace path + artifact names)
  - artifact names: `playwright-report-apex-desktop-${{ matrix.os }}`, `playwright-test-results-apex-desktop-${{ matrix.os }}`
- commit: `ci: add apex lsp playwright e2e workflow - W-22796628`
