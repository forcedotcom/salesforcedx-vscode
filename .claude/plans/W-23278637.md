### Phase 3 — review-diff: ADR-violation check + verifier proof bar

- `review-diff.js` `thermoPrompt`: add ADR check — read `docs/adr/` + in-scope `packages/*/docs/adr/`; flag diff hunks violating an accepted ADR decision as severity-graded finding. **Require the finding claim to embed `file:line` evidence pointing at the specific ADR decision it violates** (matches thermo's existing "file:line evidence required" rule), so it survives verification with a checkable citation. Prefer appending to `thermoPrompt` over a new agent (keeps fan-out count, reuses verify pass).
- **Verifier updated too.** ADR findings flow through finding-type-agnostic `verifyFindingPrompt` (review-diff.js:244-263); drop rules today (`Premise false` / `Already covered by CI` / `Zero consumer`) don't cover ADR claims. Not droppable as "just style/architecture" (verifier already forbids); not confirmed on a hallucinated citation. Add ADR-specific drop rule, parallel to `Zero consumer`:
  - **ADR premise false**: for a finding whose claim cites an ADR, the verifier must READ the cited ADR file:line and the accepted-decision text. Drop ONLY if (a) the cited ADR file/decision does not exist (hallucinated citation), (b) the ADR is not `Accepted` (e.g. Superseded/Rejected/Proposed — no binding decision to violate), or (c) the diff does not actually contradict the ADR's decision (read both; the contradiction is factually absent). Inability to prove the diff *complies* is NOT a drop — kept per the generic "when in doubt, KEEP" rule.
  - Explicitly state ADR findings inherit the unchanged generic rules otherwise (style/architecture judgment is never a drop → an ADR finding the verifier can't disprove is confirmed at claimed severity).
- Findings flow through `THERMO_SCHEMA` → verify → fix; schema/orchestration unchanged. Only the two prompt strings (`thermoPrompt`, `verifyFindingPrompt`) change.
