- Current impl: `SfCommandlet(sfProjectPreconditionChecker, PromptConfirmGatherer, OrgListCleanExecutor)`.
  - `orgList.ts` — wiring; `orgListCleanExecutor.ts` — `LibraryCommandletExecutor` calling `removeExpiredAndDeletedOrgs()` + `displayRemainingOrgs()`.
  - `orgUtil.ts` — `removeExpiredAndDeletedOrgs` uses bare `AuthInfo.listAllAuthorizations()` + `AuthRemover.create().removeAuth(...)`. `displayRemainingOrgs` uses bare `AuthInfo.listAllAuthorizations()`.
  - confirm via `PromptConfirmGatherer` (QuickPick Continue/Cancel), not modal.

## Phases

### Phase 1 — migrate listAllAuthorizations to ConnectionService

- `orgUtil.ts`: bare `AuthInfo.listAllAuthorizations()` in `removeExpiredAndDeletedOrgs` (line 264) + `displayRemainingOrgs` (line 470) → `ConnectionService.listAllAuthorizations` via `getOrgRuntime().runPromise` (Effect.fn wrapper, like `getFreshAuthorizations`).
- keep `AuthRemover.create().removeAuth` direct (no service; matches orgDelete/orgLogout).
- `removeExpiredAndDeletedOrgs` stays an async function in Phase 1, BUT note the existing outer `catch` (line 278-281) does `throw new Error(nls.localize('org_list_clean_general_error', ...))`. This plain throw is fine while the caller is the Promise-based `LibraryCommandletExecutor` (Phase 1), but MUST be removed when the caller becomes Effect (Phase 2) — see Phase 2 error-channel task. Do NOT leave a plain `throw new Error` reachable from Effect territory (SKILL.md: throw/generic Error → `Schema.TaggedError` + `Effect.fail`).
- drop now-unused `AuthInfo` import if no other use remains (still used by `getAuthFieldsFor`, `checkForSoonToBeExpiredOrgs` — keep).
- update jest mocks: `orgList.test.ts` currently mocks `AuthInfo.listAllAuthorizations` on `@salesforce/core` (lines 26-27, used at :276/:332/:363). After Phase 1, `removeExpiredAndDeletedOrgs`/`displayRemainingOrgs` resolve through `ConnectionService.listAllAuthorizations` via `getOrgRuntime().runPromise`. Decide the mock surface BEFORE asserting green: either (a) the `AllServicesLayer` wiring in the test (line 104) lets `ConnectionService.listAllAuthorizations` delegate to the real `AuthInfo.listAllAuthorizations` so the existing `@salesforce/core` mock still feeds it (verify by running just these cases and confirming the mock is hit), or (b) re-point the mocks to the `ConnectionService` layer. Land whichever the run proves correct — do not assume passthrough.
- commit: `refactor(org): use ConnectionService.listAllAuthorizations in org list clean - W-23069615`

### Phase 2 — confirm via PromptService.confirmOrThrow + Effect command

- `PromptConfirmGatherer` → `PromptService.confirmOrThrow` modal.
- new Effect.fn command `orgListCleanCommand` (model `orgDeleteDefaultCommand`, orgDelete.ts:153): get `PromptService` via `getServicesApi`, `confirmOrThrow({message: 'Confirm to continue removing deleted and expired scratch orgs', confirmLabel: nls.localize('org_list_clean_text')})`, then remove + display.
- error channel: convert `removeExpiredAndDeletedOrgs`'s outer `throw new Error(...)` into a `Schema.TaggedError` (e.g. `OrgListCleanError` with `message: Schema.String`, defined alongside the command like `OrgNotDeletableError` at orgDelete.ts:144) + `Effect.fail`. The command's error channel is then `OrgListCleanError | UserCancellationError`. If `removeExpiredAndDeletedOrgs` remains a Promise helper, wrap its call with `Effect.tryPromise({ try, catch: e => new OrgListCleanError({ message: ... }) })`; no plain `Error` may escape into the Effect.
- refresh aggregator caches: after the remove loop, call `updateConfigAndStateAggregators` (Promise) via `Effect.promise(() => updateConfigAndStateAggregators())` — matching orgDelete.ts:190 / orgLogout.ts:60. Without it the org picker (StateAggregator + ConfigAggregator) shows stale just-removed orgs. Do this BEFORE `displayRemainingOrgs` so the table reflects post-flush state.
- NO `catchAll`/`catchTag` in the command body: `registerCommandWithLayer` (registerCommand.ts:38) already does `Effect.catchTag('UserCancellationError', () => Effect.void)` and `catchAllCause → errorHandler.handleCause`. Adding `catchAll` here would be a forbidden swallow (SKILL.md anti-patterns). UserCancellationError just propagates.
- `index.ts` re-wiring (avoid dual registration):
  - REMOVE `vscode.commands.registerCommand('sf.org.list.clean', orgList)` from `registerCommands()` (index.ts:53).
  - ADD `yield* registerCommand('sf.org.list.clean', orgListCleanCommand)` inside `activateEffect` next to the existing `orgDeleteDefaultCommand`/`orgOpenCommand` registrations (index.ts:97-98), using the `registerCommandWithLayer(AllServicesLayer)` handle already created there.
- `orgListCleanExecutor.ts` + `PromptConfirmGatherer` + the old `orgList` export deleted iff unused (grep first — gatherer/executor only used by orgList).
- commit: `refactor(org): migrate org list clean confirm to PromptService.confirmOrThrow - W-23069615`
