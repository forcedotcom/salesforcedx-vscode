# W-22821055 — local/no-swallowed-rejection (Playwright)

## Context

- WI desc is placeholder; title drives scope
- Goal: ESLint rule blocks swallowed Playwright **assertion** rejections (`expect(...)` calls)
- Two sink shapes:
  1. `try { ...expect(...)... } catch { /* empty or log-only */ }`
  2. `expect(...).<chain>.catch(<empty-or-log-only>)` (await form: `await expect(x).toBeVisible().catch(() => {})`)
- Non-goal: `.catch()` on plain Playwright APIs that return `Promise<boolean>` etc. (e.g. `locator.isVisible().catch(() => false)`) — these are presence checks, not assertions. Rule must NOT flag.
- Legitimate cleanup `.catch(() => {})` on non-`expect` chains (e.g. `helpers.ts:188` `waitFor(...).catch(() => {})`) — don't flag.
- Rule key (kebab-case, ESLint config): `no-swallowed-rejection`
- Rule export (camelCase, TS): `noSwallowedRejection`
- File: `packages/eslint-local-rules/src/noSwallowedRejection.ts` (matches existing convention, see `noDuplicatePlaywrightLocators.ts` → `'no-duplicate-playwright-locators'`)

## Phases

### Phase 1 — Implement rule + tests + register

- Add `packages/eslint-local-rules/src/noSwallowedRejection.ts`
  - Export: `export const noSwallowedRejection: Rule.RuleModule = { ... }`
  - messageId: `swallowedRejection` with message: `Empty or log-only catch swallows expect() rejection. Re-throw, assert via expect.poll/toPass, or remove the assertion.`
  - No autofix (intent unclear)
  - AST visitors:
    - `TryStatement`: if `node.block` subtree contains a CallExpression whose callee is `Identifier 'expect'` (direct call, not member), AND `node.handler.body.body` is "swallowing" → report on `node.handler`.
    - `CallExpression[callee.property.name='catch']`: if the receiver chain (walk `callee.object` recursively through MemberExpression/CallExpression) contains a CallExpression whose callee is `Identifier 'expect'`, AND the first argument is "swallowing" → report.
  - Helpers:
    - `containsExpectCall(node)` — recursive walk; matches `CallExpression` where `callee.type === 'Identifier' && callee.name === 'expect'`. Uses a small manual walker (no extra deps; mirrors `noDuplicatePlaywrightLocators.ts` style).
    - `isSwallowingHandler(handlerOrFn)` returns true when:
      - **Empty body**: `BlockStatement` with `body.length === 0`, OR catch with no body, OR arrow with body `BlockStatement` length 0.
      - **Log-only body**: every statement in the block is an `ExpressionStatement` whose expression is a `CallExpression` to `console.<any>` or an Identifier matching `/^(log|logger|debug|trace)$/i` member call (e.g. `logger.warn(...)`, `console.error(...)`). No `ThrowStatement`, no `ReturnStatement`, no other call.
      - Returns false if any statement is `ThrowStatement` (re-throw), `ReturnStatement` with argument, or any non-logging call.
      - Arrow expression body (no block): treat as swallowing only if it's a literal that discards (e.g. `() => undefined`, `() => {}` — the latter is a block). `() => false` returning a boolean is NOT swallowing for our purposes when on a non-expect chain; but the chain-check already gates on expect, so for `expect(...).catch(() => false)` we DO flag (test runner ignores the rejection).
  - Edge cases documented in code comments:
    - `expect.poll(...)` and `expect(x).toPass(...)` are themselves the recommended fixes — still flagged if wrapped in swallowing catch (fine; user should remove the catch).
    - Aliased `import { expect as exp }`: out of scope; rule matches identifier name `expect` only (note in rule docs).
- Register in `packages/eslint-local-rules/src/index.ts`:
  - Import: `import { noSwallowedRejection } from './noSwallowedRejection';` (alphabetical placement after `noExportTaggedErrorInServices`)
  - Rules entry: `'no-swallowed-rejection': noSwallowedRejection,` (kebab-case key, matches existing pattern e.g. `'no-duplicate-playwright-locators': noDuplicatePlaywrightLocators`)
- Add `packages/eslint-local-rules/test/noSwallowedRejection.test.ts` (filename camelCase to match other test files in that dir; verify by `ls test/` first — adjust if convention differs)
  - **Valid cases** (must NOT flag):
    - `try { await foo(); } catch {}` — no expect
    - `await page.click().catch(() => {})` — no expect in chain
    - `await page.locator('foo').isVisible({ timeout: 400 }).catch(() => false)` — locator method, no expect (matches `helpers.ts:179-182`)
    - `await welcomeOverlay.first().waitFor({ state: 'hidden' }).catch(() => {})` — non-expect chain (matches `helpers.ts:185-188`)
    - `try { await expect(x).toBeVisible(); } catch (e) { throw new Error('wrap', { cause: e }); }` — re-throw
    - `try { await expect(x).toBeVisible(); } catch (e) { console.error(e); throw e; }` — log + re-throw
    - `await expect(x).toBeVisible({ timeout: 1000 })` — no catch at all (the right way)
  - **Invalid cases** (must flag):
    - `try { await expect(x).toBeVisible(); } catch {}` — empty catch around expect
    - `try { await expect(x).toBeVisible(); } catch (e) { /* ignore */ }` — empty body
    - `try { expect(x).toBe(1); } catch (e) { console.log(e); }` — log-only
    - `await expect(x).toBeVisible().catch(() => {})` — empty arrow on expect chain
    - `await expect(x).toBeVisible().catch(() => false)` — discarding fallback on assertion (assertion result is void; the false is meaningless and hides the failure)
    - `await expect(locator).toHaveText('foo').catch(e => logger.warn(e))` — log-only on expect chain
