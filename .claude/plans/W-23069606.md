## Phases

### Phase 1 — migrate orgLoginWebDevHub to Effect/services command

- Rewrite `src/commands/auth/orgLoginWebDevHub.ts`:
  - export `orgLoginWebDevHubCommand = Effect.fn('orgLoginWebDevHubCommand')`.
  - precondition: `api.services.ProjectService.getSfProject()` (sets `sf:project_opened`, typed err rendered by ErrorHandlerService).
  - alias: `PromptService` showInputBox (placeholder `DEFAULT_ALIAS`), undefined→`UserCancellationError`, empty→`DEFAULT_ALIAS` (mirror `authParamsGatherer` alias block).
  - build cmd `sf org login web --alias <alias> --set-default-dev-hub`; run via `TerminalService.simpleExec`. (simpleExec already injects `SF_JSON_TO_STDOUT`+`FORCE_COLOR=0` per `orgOpen.ts:91` — confirm before re-adding env; don't hardcode if redundant.)
  - verification-code modal: `showVerificationCodeIfNeeded` is `async (): Promise<void>` (`verificationCode.ts:23`); only acts when `CODE_BUILDER_STATE` set, else no-op. Inside the `Effect.fn` generator, bridge it as `yield* Effect.promise(() => showVerificationCodeIfNeeded())` — NOT the old `void showVerificationCodeIfNeeded()` fire-and-forget (`orgLoginWebDevHub.ts:52`) which escapes the runtime and swallows the modal's errors (SKILL anti-pattern). DECISION: yield it immediately BEFORE the `simpleExec` call (modal shows the code, user dismisses, then exec/browser auth proceeds — preserves old "code visible for the auth flow" intent). `Effect.promise` (not `Effect.tryPromise`) because the util never rejects in practice; if review wants defect-safety, `Effect.tryPromise` mapping to a tagged error is the upgrade path. Document the ordering in a code comment.
  - on success: `yield* Effect.promise(() => updateConfigAndStateAggregators())` (it is `async (): Promise<void>`, `orgUtil.ts:115`).
  - drop `SfCommandlet`/`SfCommandletExecutor`/`CliCommandExecutor`/`ParametersGatherer`/`sfProjectPreconditionChecker` imports.
- `index.ts`: drop `orgLoginWebDevHub` from barrel import + remove `vscode.commands.registerCommand('sf.org.login.web.dev.hub', ...)`; add `yield* registerCommand('sf.org.login.web.dev.hub', orgLoginWebDevHubCommand)` next to orgOpen; import from `./commands/auth/orgLoginWebDevHub`.
- `src/commands/index.ts`: drop `orgLoginWebDevHub` re-export (now imported directly).
- constant: add `ORG_LOGIN_WEB_DEV_HUB = 'sf.org.login.web.dev.hub'` to `constants.ts`, use in index (parity w/ `ORG_OPEN_COMMAND`).
