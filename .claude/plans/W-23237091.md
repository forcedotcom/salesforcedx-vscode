# W-23237091 — Migrate sf.org.login.access.token to services (CLI/simpleExec)

## Context

- Supersedes W-23069607 (closed PR #7622) — that used in-process `AuthInfo.create`. This redo shells to sf CLI.
- Current: `packages/salesforcedx-vscode-org/src/commands/auth/orgLoginAccessToken.ts` — `LibraryCommandletExecutor` + `SfCommandlet`, `AuthInfo.create({accessTokenOptions})` + `handleAliasAndDefaultSettings`, bad-oauth special-case, then `updateConfigAndStateAggregators()`.
- Target: full Effect command (pattern: `orgOpenCommand` / `orgDeleteDefaultCommand` in `commands/orgOpen.ts`, `commands/orgDelete.ts`), registered via `registerCommandWithLayer(AllServicesLayer)` in `index.ts`.
- Token via `SF_ACCESS_TOKEN` env (core recognizes it — `node_modules/@salesforce/core/lib/config/envVars.js:63`); plugin-auth reads it pre-prompt → headless. Never on argv/span/history. `TerminalService.simpleExec` already supports `env` + annotates env keys only, not values (`terminal/terminalService.ts:44-46`).
- CLI cmd: `sf org login access-token --instance-url <url> --alias "<alias>" --no-prompt`. Alias regex-validated (rejects shell metachars) → quoted arg injection-safe.
- Errors: `TerminalServiceError` → `ErrorHandlerService.handleCause` toast (CLI's own msg) via `registerCommand.ts:39`. No bad-oauth special-case, no new i18n key. Drop now-unused `org_login_access_token_bad_oauth_token_message`.
- `isAlphaNumSpaceString` lives in `orgCreate.ts:32` (`/^\w+( *\w*)*$/`). Extract to org-pkg shared util; wire orgCreate + access-token alias prompt. No backfill other gatherers.
- Keep `AccessTokenParamsGatherer` for prompt collection (instanceUrl/alias/token); command consumes its `ContinueResponse` then execs. Keep `updateConfigAndStateAggregators()` post-exec.
- Project guard: current cmd uses `sfProjectPreconditionChecker` (`orgLoginAccessToken.ts:57`) + `package.json` palette gate `when: sf:project_opened` (`package.json:221`). Access-token login does NOT need a project root. DROP the guard in both places: new Effect command omits `getSfProject()` (matches `orgDeleteDefaultCommand`, which has no project check); ALSO remove the `when: sf:project_opened` clause from the `commandPalette` entry. Mismatching them (keep palette gate, drop code guard or vice versa) is a UX regression — they must move together.
