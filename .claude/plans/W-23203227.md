### Phase 3 — wire into no-org reactor branch (after Phase 0 rebase)

`watchers/testDiscovery.ts`: in the no-org (`else`) branch of the ternary, chain after `clearAllTestItems()`:
`Effect.promise(() => testController.clearAllTestItems())` → `Effect.zipRight(Effect.promise(() => testController.closeAllApexTestingTabs()))` → `Effect.zipRight(ApexTestDiscoveryService.clearAll)`. Sequence: tree clear → tabs close → VFS purge.

**Replace the blanket `Effect.catchAll` + `console.debug`** (forbidden per effect anti-patterns: `catchAll` swallowing, `console.*` in Effect pipeline). The only typed error in the combined pipeline is `DiscoveryClearError` — `pruneForeignOrgClasses` (discover branch) and the new `clearAll` (no-org branch) both fail with it; `refresh`/`clearAllTestItems`/`closeAllApexTestingTabs` are `Effect.promise` (`never` error channel). Use:

```ts
Effect.catchTag('DiscoveryClearError', error =>
  Effect.logDebug('Apex Testing: discovery VFS clear failed').pipe(
    Effect.annotateLogs({ orgKey: error.orgKey, scheme: 'apex-testing' })
  )
)
```

(intentional ignore — VFS purge failure must not break the org-change reactor — but scoped to the one type and structured-logged, not swallowed blind). If a future op adds a second typed error, widen to `Effect.catchTags({...})` then.
