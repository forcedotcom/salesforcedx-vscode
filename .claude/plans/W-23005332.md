# W-23005332 — consolidate soql QueryResult & DescribeSObjectResult types

## Context

soql re-derives query/describe types in 3 files. Violates ADR 0005 (services API = contract) + DRY.

Duplicates (verified):
- `Awaited<ReturnType<Connection['query']>>` 3x: `src/types.ts:9` (generic `QueryResult<T>`), `src/commands/dataQuery.ts:15` (non-generic), `src/editor/queryRunner.ts:20` (non-generic).
- `src/types.ts:12` local `DescribeSObjectResult` = `Awaited<ReturnType<Connection['describe']>>`. services already exports `DescribeSObjectResult` (`salesforcedx-vscode-services/src/index.ts:194`, def in `core/transmogrifierService.ts:15`, same underlying `Awaited<ReturnType<Connection['describe']>>`). soql already deps services (package.json:71) + imports types from it (codeCompletion.ts:14 — type-only).

Consumers:
- `QueryResult<JsonMap>`: queryDataViewService, queryDataFileService, queryDataHelper, soqlEditorInstance.
- `QueryResult['records']` (non-generic): dataQuery (buildFlattenedGridModel, generateTableOutput, convertToCSV, getFlattenedSoqlGridPayload).
- non-generic `QueryResult` param: dataQuery (saveResultsToCSV, displayTableResults, convertQueryResultToCSV), queryRunner (runQuery return).
- `DescribeSObjectResult`: soqlEditorInstance (4x), sourced from `MetadataDescribeService.describeCustomObject` (services).

Collapse strategy: generic stays in types.ts. Non-generic call sites → `QueryResult<JsonMap>`. queryRunner records = flattened `JsonMap[]` (line 50-52); dataQuery records consumed as `unknown` via `isRecord` guards — `JsonMap` arg compatible. `JsonMap` already imported in queryRunner:10 + soqlEditorInstance:10; add import in dataQuery.

Constraints: type-only consolidation, no runtime behavior change; no Effect Schema downgrade; no new runtime dep on @salesforce/core or jsforce (type-only OK, ADR 0008). services `DescribeSObjectResult` is type-only export → safe.

Pattern ref: PR #7457.

## Phases

### Phase 1 — types.ts: drop local DescribeSObjectResult, re-export from services

- `src/types.ts`: remove line 12 local `DescribeSObjectResult`.
- add `export type { DescribeSObjectResult } from 'salesforcedx-vscode-services';` (type-only).
- keep generic `QueryResult<T>` (still derives from `Connection['query']` — sole remaining `Awaited<ReturnType<Connection['query']>>` in soql src, satisfies verify).
- soqlEditorInstance.ts already imports `DescribeSObjectResult` from `../types` (line 9) — unchanged, now transitively from services.

commit: `refactor(soql): re-export DescribeSObjectResult from services instead of re-deriving - W-23005332`

### Phase 2 — collapse non-generic QueryResult dupes onto types.ts generic

- `src/commands/dataQuery.ts`: remove local `type QueryResult` (line 15) + `import type { Connection }` (line 7, now unused). import `QueryResult` from `../types`. import `JsonMap` from `@salesforce/ts-types`. Every reference to the now-generic `QueryResult` must supply the type arg `JsonMap` — bare `QueryResult` won't compile (generic requires 1 type arg). Two forms:
  - bare param annotations → `QueryResult<JsonMap>`: lines 41 (`saveResultsToCSV`), 564 (`displayTableResults`), 578 (`convertQueryResultToCSV`).
  - indexed access → `QueryResult<JsonMap>['records']` (NOT bare `QueryResult['records']`): lines 118, 134, 148 (`generateTableOutput`), 487 (`convertToCSV`).
- `src/editor/queryRunner.ts`: remove local `type QueryResult` (line 20). import `QueryResult` from `../types`. `runQuery` return `Promise<QueryResult>` → `Promise<QueryResult<JsonMap>>` (line 27). drop `import type { Connection }`? — still used for `conn: Connection` param (line 23) → keep.
- verify `flattenQueryRecords` output (`JsonMap[]`) assignable to `QueryResult<JsonMap>['records']`.

commit: `refactor(soql): import shared QueryResult instead of local re-declarations - W-23005332`
