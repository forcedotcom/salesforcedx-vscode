# W-23127989 — E2E flake: Apex Test Suite (trace ENOENT + palette timeout)

## Context

- Retry rate 57%; 2 distinct flakes:
  1. **Trace ENOENT** (dominant, not real fail): `ENOENT .playwright-artifacts/*.zip` during per-test trace cleanup. Electron + per-test trace bundling. PW 1.45 had this bug (fixed 1.45.1 PR #31437); recurs on 1.60.0 under load.
  2. **Command-palette 30s timeout** on run-suite step.
- Desktop trace = `'on'` (every test bundles zip) — `packages/playwright-vscode-ext/src/config/createDesktopConfig.ts:34`. Web uses `'on-first-retry'` (`createWebConfig.ts:34`).
- `createDesktopConfig` shared by 14 packages → scope change via opt-in option, not global flip.
- `executeCommandWithCommandPalette` already self-retries `toPass({timeout: 30_000})` (`pages/commands.ts:140-148`). Run-suite cmd (`apexTestSuite.headless.spec.ts:163`) can race extension-command registration → readiness wait fixes.
- `verifyCommandExists` opens its own palette and **closes it** before returning (`commands.ts:209` via `retryCommandPaletteSearch`); it does NOT leave a palette open for a following `executeCommandWithCommandPalette`. Both helpers open fresh palettes — the gate's value is "command is registered," not "palette already open." Sequence: `verifyCommandExists` → `executeCommandWithCommandPalette` (latter opens fresh palette; its own 30s `toPass` retry then succeeds immediately since the cmd is registered).
- Toggle-panel (`:172`) already wraps in `toPass({timeout: 30_000})` — no separate robustness change; WI cites `:163` run-suite step, not `:172`.
- Spec: `apexTestSuite.headless.spec.ts`. WI cites `:163` run-suite step (within `:169-172` verify block).

## Phases

### Phase 1 — opt-in trace mode in shared desktop config

- Add optional `trace?: 'on' | 'on-first-retry' | 'retain-on-failure' | 'off'` to `DesktopConfigOptions`.
- Default `'on'` (unchanged for other 13 packages).
- Wire `use.trace: options.trace ?? 'on'`.
- Files: `packages/playwright-vscode-ext/src/config/createDesktopConfig.ts`
- Commit: `test(e2e): allow per-shard trace mode in desktop playwright config - W-23127989`

### Phase 2 — disable per-test trace bundling for apex-testing shard

- Pass `trace: 'on-first-retry'` in apex-testing desktop config → only bundles on retry, eliminates per-test zip churn driving ENOENT.
- Files: `packages/salesforcedx-vscode-apex-testing/test/playwright/playwright.config.desktop.ts`
- Commit: `test(apex-testing): trace on-first-retry to fix ENOENT flake - W-23127989`

### Phase 3 — readiness wait before run-suite command

- Insert before run-suite `executeCommandWithCommandPalette` (`:163`): `verifyCommandExists(page, packageNls.apex_test_suite_run_text, 30_000)`. Gate confirms the extension command is registered; it opens+closes its own palette (`commands.ts:209`), so the following `executeCommandWithCommandPalette` opens a fresh palette and its built-in 30s `toPass` retry succeeds immediately.
- No toggle-panel change (`:172` already has 30s `toPass`; not cited by WI). If `:172` flakes with its own evidence later, that is a separate phase.
- Files: `packages/salesforcedx-vscode-apex-testing/test/playwright/specs/apexTestSuite.headless.spec.ts`
- Commit: `test(apex-testing): readiness wait before run-suite palette cmd - W-23127989`
