# W-23145031 — SOQL Builder: ALL ROWS support

## Context

- Goal: query deleted/archived records from SOQL Builder via `ALL ROWS`.
- Vendored ANTLR parser (`@salesforce/soql-common/soql-parser.lib`) lacks ALL/ROWS tokens → must strip before parse, re-inject on serialize.
- SOQL document text = source of truth. UI checkbox → `allRows` flag in UiModel → `convertUiModelToSoql` appends `ALL ROWS` → round-trips back through `deserialize`.
- Execution: jsforce `conn.query(soql, {scanAll:true})` routes to `/queryAll` endpoint (`node_modules/@jsforce/jsforce-node/lib/query.js:368`). `/queryAll` rejects literal `ALL ROWS` text → strip before send.
- `conn` is `@salesforce/core` Connection extends jsforce Connection (`node_modules/@salesforce/core/lib/org/connection.d.ts:34`) — `query` opts include `scanAll`.

### Two independent execution paths (both must handle ALL ROWS)

1. `editor/queryRunner.ts` `runQuery` — SOQL Builder webview "Run Query" button. Calls `conn.query(pureSOQLText, {...})`.
2. `commands/dataQuery.ts` `runSoqlQuery` — code-lens "Run Query" + command-palette (`data_query_document_text`, `data_query_selection_text`). Calls `connection.query(query, {...})` (line 36) and `connection.tooling.query(query, {...})` (line 35). This path is what `soql-run-query.spec.ts` exercises. Outside the original file list — adding it now.

Both paths: strip trailing `ALL ROWS` (case-insensitive), pass `{ scanAll: true }`. Tooling branch same — jsforce Tooling extends same query opts, rejects literal text too.

### Shared helper (avoid duplicating strip/detect logic across 2 paths)

- New `editor/allRows.ts` exporting `const stripAllRows = (soql: string): { soql: string; scanAll: boolean }`.
  - regex `/\s+ALL\s+ROWS\s*$/i` on the comment-stripped SOQL text; returns stripped text + `scanAll` bool.
  - functional, single return, no `let`.
- `queryRunner.ts` and `dataQuery.ts` both import it.

## Phases

> **ONE PR (atomic).** Phase 1 serializes `allRows`; Phase 2 sets it via UI. Until Phase 2, field falsy → `ALL ROWS` omitted (safe). Phase 1 tests construct `QueryImpl` directly — no UI round-trip dep. Don't merge Phase 1 standalone.

### 1. soql-model: `allRows` on Query + serialize + parse strip/inject

- `packages/soql-model/src/model/model.ts`: add `allRows?: boolean` to `Query` type.
- `packages/soql-model/src/model/impl/queryImpl.ts`:
  - add `public allRows?: boolean;` instance field (property, not ctor arg — keeps ctor signature stable).
  - `toSoqlSyntax`: after the `update` clause (last block before `return`), append `${' '.repeat(opts.indent)}ALL ROWS\n` when `this.allRows`.
- `packages/soql-model/src/serialization/deserializer.ts`:
  - strip trailing `ALL ROWS` (regex `/\s+ALL\s+ROWS\s*$/i`) from the header-padded SOQL text BEFORE `parser.parseQuery`; track stripped bool.
  - after `QueryListener` builds the `QueryImpl`, set `query.allRows = true` when stripped (property assignment on returned obj, not via ctor — mirrors how `headerComments` is attached).
- Unit tests:
  - `deserializer.test.ts`: parse `SELECT ... ALL ROWS` → `allRows === true`, no parse errors; parse without ALL ROWS → `allRows` falsy.
  - `queryImpl.test.ts`: construct `QueryImpl` then set `.allRows = true` → `toSoqlSyntax` emits `ALL ROWS` as last clause; `.allRows` unset → not emitted.
- commit: `feat(soql-model): ALL ROWS parse + serialize - W-23145031`

### 2. UI service round-trip: carry `allRows` through UiModel

- `packages/salesforcedx-vscode-soql/src/soql-builder-ui/modules/querybuilder/services/model.ts`: add `allRows?: boolean` to `ToolingModelJson`.
- `services/soqlUtils.ts`:
  - `convertSoqlModelToUiModel`: read `queryModel.allRows` → set `allRows` on the returned tooling model (next to `headerComments` at the `toolingModelTemplate` object literal there, ~line 110).
  - `convertUiModelToSoqlModel`: after constructing `queryModel = new QueryImpl(...)` (~line 209-217), set `queryModel.allRows = uiModel.allRows` (mirrors the `queryModel.headerComments = ...` pattern at line 218-220). Works because Phase 1 made `toSoqlSyntax` read `this.allRows`.
- `services/toolingModelService.ts`:
  - **`allRows: false` in `toolingModelTemplate` required** (toolingModelService.ts:18-27) — `setSObject` resets via spread (`changeModel({ ...toolingModelTemplate, ... })`, ~line 93); omitting leaves `undefined` → inconsistent round-trip.
  - add setter `setAllRows(allRows: boolean): void` via `changeModel`; include in the service return / `ModelProps`.
- Unit test: `soqlUtils.test.ts` round-trip `allRows` true and false; assert `toolingModelTemplate` default is `false` (not undefined).
- commit: `feat(soql-soql-builder): carry ALL ROWS thru UI model - W-23145031`

### 3. UI checkbox + e2e

- `messages/i18n.ts`: add `label_all_rows: 'Include deleted/archived records'`.
- `app/app.html`: add checkbox (mirror limit section markup; `data-el-all-rows`, `checked={query.allRows}`, `onchange={handleAllRowsChanged}`) inside `showQueryBuilderForm` after limit.
- `app/app.ts`: `handleAllRowsChanged(e)` → `this._modelService?.setAllRows(e.target.checked)`.
- **e2e (in scope this phase):**
  - `test/playwright/specs/soql-builder.spec.ts`: add a step after the existing limit/where/orderBy preview steps — `soqlFrame.locator('[data-el-all-rows]')` check → assert `soqlFrame.locator('.query-preview-container pre')` text contains `ALL ROWS`; uncheck → assert it no longer contains `ALL ROWS`. Use existing `soqlFrame` + `expect(...).toContainText` / `not.toContainText` pattern already in that file.
  - `test/playwright/specs/soql-run-query.spec.ts`: add a step that types a query ending in `ALL ROWS`, runs it (code-lens REST path — the dataQuery.ts path), asserts query completes.
    - **No `waitForRequest`:** network interception works only in web/renderer, not desktop/Electron extension host (Node HTTP call invisible to Playwright; spec runs both modes). Durable cross-mode signal: `QUERY_COMPLETE_TEXT` wait — org rejects literal `ALL ROWS` on `/queryAll`, so "records returned" proves strip+route worked.
- commit: `feat(soql-soql-builder): ALL ROWS checkbox + e2e - W-23145031`

### 4. Execution route to queryAll (both paths) + shared helper

- New `editor/allRows.ts`: export `stripAllRows(soql)` (see Context). Unit-tested in isolation.
- `editor/queryRunner.ts` `runQuery`: after `pureSOQLText = soqlComments.parseHeaderComments(...).soqlText`, run `const { soql, scanAll } = stripAllRows(pureSOQLText)`; call `conn.query(soql, { autoFetch: true, maxFetch: ..., scanAll })`.
- `commands/dataQuery.ts` `runSoqlQuery`: apply `stripAllRows(query)` and pass `scanAll` into BOTH branches — `connection.tooling.query(soql, { autoFetch, maxFetch, scanAll })` (line 35) and `connection.query(soql, { autoFetch, maxFetch, scanAll })` (line 36). Keep the `Effect.fn` shape; this is a pure transform before the existing `Effect.promise`.
- Unit tests:
  - `allRows.test.ts`: trailing `ALL ROWS` (varied casing/whitespace) → stripped + `scanAll:true`; absent → unchanged + `scanAll:false`; `ALL ROWS` not at end (e.g. in a string literal) → NOT stripped.
  - `queryRunner` test: ALL ROWS present → `conn.query` called with `scanAll:true` + stripped text; absent → `scanAll:false` + unchanged.
  - `dataQuery` test: REST + Tooling branches each pass `scanAll:true` + stripped text when ALL ROWS present.
- commit: `feat(soql): route ALL ROWS query to queryAll (builder + code-lens/palette) - W-23145031`
