# W-23196265 — Stabilize windows-latest desktop E2E page fixture

## Context

- Recurring **windows-latest desktop** E2E failures on develop (~daily). NOT macOS org issue.
- Signature (every suite, win-only):
  - `Test timeout of 60000ms exceeded while setting up "page".`
  - `page.setViewportSize: Target page, context or browser has been closed`
  - origin: `createDesktopTest.ts` page fixture (`firstWindow()` → `setViewportSize` → `waitForSelector(WORKBENCH, 60s)`)
- Root cause: `electronApp.firstWindow()` window closes/never opens before WORKBENCH resolves. Fixture-level, not test bodies.
- **Budget bug (key):** plain test-scoped fixture (createDesktopTest.ts:437), no per-fixture timeout → setup counts against test-level timeout (60s, createDesktopConfig.ts:39). launch (60s) + firstWindow + WORKBENCH wait (60s) share 1 budget. Retry inside fixture compounds the timeout.
- Co-symptoms: `EPERM rmdir @unrs/resolver-binding-wasm32-wasi`; `Try E2E timed out after 5min`. Win launch/file-handle contention.
- Relevant files (verified paths):
  - `packages/playwright-vscode-ext/src/fixtures/createDesktopTest.ts` (electronApp launch 377-386; teardown 388-433; page fixture 437-465)
  - `packages/playwright-vscode-ext/src/config/createDesktopConfig.ts` (per-test timeout 39)
  - `packages/playwright-vscode-ext/src/utils/helpers.ts:433` `isWindowsDesktop()` (`VSCODE_DESKTOP==='1' && platform==='win32'`) — reuse in fixture; NOT in config (see Phase 2).
- Playwright fixture timeout: tuple form `[fn, { scope: 'test', timeout }]` (test.d.ts:6687,6691) — independent of test-level timeout.
- Playwright `firstWindow()` semantics: returns **same closed Page** on a 2nd call — not a new window. Retry = stale-page trap. New window: `waitForEvent('window')` or relaunch.
- No prior win32 launch-retry attempt in history.

## Phases

> Phase 1 + Phase 2 must land **together** (single PR). Phase 1's relaunch/retry only works if the `page` fixture has its own timeout budget; that budget is added in Phase 1 itself (per-fixture timeout), and Phase 2 raises the *test-level* budget so the rest of the test body still has room. Splitting Phase 1 (retry) and Phase 2 (budget) into sequential commits leaves a window where retry eats the 60s test budget and still fails — do not ship Phase 1 alone.

### Phase 1 — Harden page fixture: independent budget + relaunch (not firstWindow retry)

`createDesktopTest.ts`:

- **Give the `page` fixture its own timeout (Finding: budget).** Convert the `page` fixture to the tuple form:
  `page: [async ({ electronApp }, use) => { ... }, { scope: 'test', timeout: isWindowsDesktop() ? 180_000 : 90_000 }]`.
  Verify the tuple typechecks against `TestFixtures` (test.d.ts:6687/6691 — `timeout?: number` allowed on test-scoped fixtures).
- **Do NOT retry `firstWindow()` (Finding: stale page).** `firstWindow()` returns the same (possibly closed) Page on a second call. Instead, build readiness around a fresh window:
  - Extract launch logic (377-386) into `launchElectron(launchArgs, vscodeExecutable, videosDir): Promise<ElectronApplication>`. Keep `timeout: 60_000`.
  - Add `waitForWorkbenchWindow(electronApp, { workbenchTimeout }): Promise<Page>`:
    1. `const page = await electronApp.firstWindow()`.
    2. `await page.waitForLoadState('domcontentloaded')`.
    3. `await page.setViewportSize(...)` inside try; on `closed`/`Target ... has been closed` → throw a named `WindowsDesktopLaunchInstabilityError`.
    4. `await page.waitForSelector(WORKBENCH, { timeout: workbenchTimeout })`; on timeout → same named error.
  - On win32 (`isWindowsDesktop()`) instability: **relaunch the whole electron app** (close best-effort, `launchElectron(...)` again) up to 1 retry, then re-run `waitForWorkbenchWindow` on the *new* app instance. Do not call `firstWindow()` twice on the same instance. Because the `electronApp` fixture owns the app handle, relaunch happens by re-invoking `launchElectron` and swapping the reference used by the `page` fixture (move launch+relaunch into a small ready-app helper consumed by the `page` fixture, or expose a `relaunch` callback from the `electronApp` fixture). Keep the relaunched app under the same teardown (kill + temp cleanup).
  - If retries exhausted, throw `WindowsDesktopLaunchInstabilityError` (clean, named) → triggers Playwright `retries: 2` (CI) at the test level. This is the safe fallback: a full test retry gets a fresh worker + fresh app, avoiding stale-page state.
- Import `isWindowsDesktop` from `../utils/helpers`.

commit: `fix(playwright): win32 desktop page fixture — own timeout budget + relaunch on launch instability - W-23196265`

### Phase 2 — Longer test-level budget on win32 (config), preserving caller override

`createDesktopConfig.ts`:

- **Preserve `options.timeout` (Finding: lost override).** The win32 bump applies ONLY when the caller did not pass `timeout`. Exact expression:
  `timeout: process.env.DEBUG_MODE ? 0 : (options.timeout ?? (process.platform === 'win32' ? 120_000 : 60_000))`.
  Do NOT substitute the win32 ternary into the value position — keep it strictly inside the `??` default branch.
- Note: use `process.platform === 'win32'` here, NOT `isWindowsDesktop()` — `isWindowsDesktop()` also requires `VSCODE_DESKTOP==='1'`, which is a job env var, not guaranteed at config-load time; `process.platform` is the reliable signal in the config module.
- Optionally raise `actionTimeout` win32 15s → 20s (same override-safe pattern is N/A here; it's a fixed config value).

commit: `fix(playwright): longer test-level setup budget on win32 desktop, override-safe - W-23196265`

files:
- `packages/playwright-vscode-ext/src/config/createDesktopConfig.ts`

### Phase 3 — Reduce win file-handle contention (teardown EPERM)

`createDesktopTest.ts` teardown (388-433):

- EPERM on `fs.rm` of userDataDir = `@unrs/resolver-binding-wasm32-wasi` handle held post-kill. Already `maxRetries:10, retryDelay:200`. Bump win32 → `maxRetries:20, retryDelay:300`.
- After win32 `process.kill(SIGKILL)` (419-423), await a short proc-exit Promise (`proc.on('exit')` race with timeout) before `fs.rm` so handles release first.

commit: `fix(playwright): await win32 electron exit before temp cleanup to avoid EPERM - W-23196265`
