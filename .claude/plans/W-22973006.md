# W-22973006 — Refactor getRetrievedFileUri to use SDR RetrieveResult

- arg = success value of `MetadataRetrieveService.retrieve(...)`; success channel = `RetrieveResult` (services/core/metadataRetrieveService.ts:146 `pollStatus()` -> `RetrieveResult`; :177 returns outcome)
- caller (testController.ts:1100) narrows out `typeof result === 'string'` cancellation branch before :1105 call -> `result` already `RetrieveResult`

- test mock returns `{ getFileResponses: () => [] }` cast (testController.test.ts:18); `getRetrievedFileUri` module-private — retype won't break callers

## Coverage reality (advocate correction)

- ZERO e2e (Playwright/WDIO) coverage exists for `sf.apex.test.orgOnlyClass.retrieve`. Prior plan claimed "retrieve-open flow covered by orgOnlyClass e2e" — FALSE.
- User-visible flow (test explorer org-only class -> codelens "retrieve" -> SDR retrieve -> open retrieved `.cls`) exercised ONLY by mocked unit tests (testController.test.ts mock returns `{ getFileResponses: () => [] }`).
- `getRetrievedFileUri` arg is the success value of `MetadataRetrieveService.retrieve` = SDR `RetrieveResult` produced at RUNTIME by `pollStatus()` (services/core/metadataRetrieveService.ts:146). Static typing narrows compile-time contract; does NOT prove runtime object matches across SDR version bumps (root SDR `^12.32.5`, currently 12.35.10 — floating).
- Decision: strong-type signature AND retain runtime guards as genuine defense-in-depth (not "cheap but removable"). Add the missing Playwright desktop e2e so the flow has real coverage independent of the type change. Both, per finding option (1)+(2).

### Phase 1 — refactor with retained runtime defense (single commit)

- package.json: add `@salesforce/source-deploy-retrieve` to devDependencies (type-only, erased at build); root install
- typeGuards.ts: delete unused `MetadataRetrieveFileResponse`, `MetadataRetrieveOutcomeLike`; KEEP `isPlainObject`; re-evaluate `isMetadataRetrieveOutcomeLike` / `isMetadataRetrieveFileResponse`: keep them (still used as runtime defense below) — only delete a guard if it has zero remaining call sites after refactor.

  - imports: add `import type { RetrieveResult, FileResponse } from '@salesforce/source-deploy-retrieve'`; drop only typeGuards symbols that become genuinely unused
  - retype `getRetrievedFileUri(result: RetrieveResult): URI | undefined` (compile-time contract via SDR types)
  - body: iterate `result.getFileResponses()` typed `FileResponse[]`; `.find` first non-empty `filePath` -> `URI.file`

Rationale: value from runtime `pollStatus()`; SDR is floating dependency; caller treats path as best-effort. Malformed shape → `undefined` (skip editor open), not throw. Express the per-item filtering with strong types: `FileResponse[]` `.find(r => typeof r.filePath === 'string' && r.filePath.length > 0)` rather than the old `isMetadataRetrieveFileResponse` hand guard — keeps defense while using SDR types.
