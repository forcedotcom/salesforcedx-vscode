# W-22916761 — Remove legacy WDIO Apex LSP E2E + add hover Playwright coverage

## Context

Legacy WDIO `apexLsp.e2e.ts` (7 cases: indexing, go-to-def, autocompletion, 4 restart variants) redundant with Playwright `apexLsp.desktop.spec.ts` + `apexLspRestart.desktop.spec.ts`. Workflow references in `apexE2E.yml` and `triggerE2EForCommit.yml` still invoke it. Hover assertion: sync gate only (waits for `ExampleClass` before Ctrl+Click) — not validating signature content.

## Phases

### Phase 1 — Delete legacy WDIO spec + all workflow dispatch/job references

- Delete `packages/salesforcedx-vscode-automation-tests/test/specs/apexLsp.e2e.ts`
- `apexE2E.yml`: remove `apexLsp` input (both `workflow_dispatch` + `workflow_call`), remove `apexLSP` job (lines 69-76). No caller passes `apexLsp` explicitly — `e2e.yml` calls `apexE2E.yml` without that input, relying on default. Removing the input + job is safe; GitHub Actions ignores unrecognized inputs from callers that never sent them.
- `triggerE2EForCommit.yml`: remove `apex-lsp` job (lines 37-43) — this job calls `runE2ETest.yml` with `testToRun: 'apexLsp.e2e.js'`; after the spec file is deleted, leaving this job would cause CI failure on every commit-triggered run.

**Verified:** `grep -r "apexLsp" .github/workflows/` shows no explicit `apexLsp:` input passed by any caller workflow. Only `apexE2E.yml` defines and consumes it internally.

**Files:**
- `packages/salesforcedx-vscode-automation-tests/test/specs/apexLsp.e2e.ts`
- `.github/workflows/apexE2E.yml`
- `.github/workflows/triggerE2EForCommit.yml`

**Commit:** `fix: remove legacy WDIO apexLsp.e2e.ts and workflow references - W-22916761`

### Phase 2 — Add `apexLspHover.desktop.spec.ts`

Mirrors `lwcLspHover.headless.spec.ts` pattern:
- Open `ExampleClass.cls`, wait for LSP ready
- Hover `SayHello` token
- Assert `.monaco-hover` contains LSP-provided method signature (`String` return type and `SayHello(String name)`)
- Uses `desktopFixtures` (ExampleClass pre-seeded), `waitForApexLspReady`
- Console monitoring + `validateNoCriticalErrors`

**Files:**
