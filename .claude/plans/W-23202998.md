# W-23202998 — Apex Testing view: clear on logout/delete default org

## Context

- Repro: SFDX Delete/Log Out Default Org → Apex Testing sidebar keeps org-discovered tests. Needs window reload to reach no-org state (empty apex tree; LWC unaffected).
- Root cause: `watchers/testDiscovery.ts` subs `TargetOrgRef.changes` → `Stream.map(org => org.orgId)` → `Stream.filter(isString)`. Logout sets `orgId` undefined → filtered out → `refresh()` never fires → tree stale.
- Fix: handle `orgId` → undefined transition. Clear apex tree + skip org discovery. Same result as reload, no reload.
- Scope: apex-testing ext only. LWC tests local/in-workspace, unaffected. Pre-existing, not from Effect refactor PR.
- `clearTestItems` is `private` (testController.ts:854). Need public entry. `refresh()` (line 123) already does `invalidateConnection` + `clearTestItems` + discover — too much (it discovers). Need clear-only public method.
- Watcher receives `ApexTestController` instance (index.ts:65, `getTestController()`).
- `TargetOrgRef` is a `SubscriptionRef<{orgId?: string; ...}>`. `.changes` emits current value (element 0) then every future `SubscriptionRef.set`. NOT a PubSub — test harness must drive it with `SubscriptionRef.make` + `SubscriptionRef.set`, never PubSub.publish.
- No existing test for `watchers/testDiscovery.ts`. Harness reference: `test/jest/watchers/apexMetadataChangeWatcher.test.ts` (Layer.succeed(ExtensionProviderService) + forkScoped + TestContext) — but that one uses PubSub because `MetadataChangeNotificationService` exposes a PubSub. Do NOT copy the PubSub; substitute a `SubscriptionRef` for the TargetOrgRef stream source.

## Adversarial concerns (must address in code)

- `clearAllTestItems` must reset `this.hasRestoredResults = false` (testController.ts:130 does this in `refresh()`; testController.ts:228-230 skips `restorePreviousResults` while `hasRestoredResults` is true). Without it, logout→reconnect on a new org never restores that org's prior results.
- Race: discovery may be in-flight (`discoveryInProgress`, testController.ts:193-202 runs to completion) when logout fires. Synchronous `clearTestItems()` would be re-populated by the resolving discovery. `clearAllTestItems` must await `discoveryInProgress` before clearing (mirrors `refresh()` testController.ts:124-126 which awaits it first), so the clear lands after any in-flight discovery completes.

## Phases

### Phase 1 — public clear method on ApexTestController

- testController.ts: add `public async clearAllTestItems(): Promise<void>` that:
  1. `if (this.discoveryInProgress) await this.discoveryInProgress;` — drain in-flight discovery first (avoids race repopulating the tree).
  2. `this.invalidateConnection();` — drop connection/testService, clear org apex class cache.
  3. `this.clearTestItems();` — empty tree + suite/class/method maps.
  4. `this.hasRestoredResults = false;` — so next re-auth discovery restores the new org's results.
- Mirrors no-org reload state. Keep private `clearTestItems` as-is (still used by refresh/discover).
- commit: `fix(apex-testing): expose clearAllTestItems for no-org state - W-23202998`

### Phase 2 — watcher handles orgId→undefined

- watchers/testDiscovery.ts: drop `Stream.filter(isString)`. Map to `org.orgId` (string | undefined), `Stream.changes` on full value (incl undefined transition).
- `Stream.runForEach`: branch —
  - orgId string → existing channel log + `testController.refresh()` (org discovery).
  - orgId undefined → `testController.clearAllTestItems()`, skip discovery (no refresh).
- Replace the forbidden guard at testDiscovery.ts:30-33 (`Effect.catchAll` + `console.debug` + `Effect.void` swallow). Use:
  `Effect.catchAll(err => Effect.log('[Apex Testing] Test discovery setup failed', { error: err }))`
  — logs via Effect.log (not console.debug) and surfaces the failure to the log instead of silently swallowing to `Effect.void`. Prefer `catchTag` on the specific discovery error type if one is exposed; fall back to `catchAll`+`Effect.log` only if the failure channel is unrefined.
- Remove now-unused `isString` import.
- commit: `fix(apex-testing): clear test tree on logout/delete default org - W-23202998`
