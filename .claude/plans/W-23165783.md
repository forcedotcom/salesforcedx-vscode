## Design decisions

- New `views/apexTestTreeService.ts` (or `services/`): `Effect.Service` `ApexTestTreeService`, `accessors:true`, `dependencies:[]` (reach ambient services via `api.services`).
- Owns Refs: `suiteItems`/`classItems`/`methodItems`/`classToParentItem` (`Ref<Map>`), `hasRestoredResults`/`isRestoringResults` (`Ref<boolean>`), plus the in-flight `Ref<Option<Deferred<void, DiscoveryError>>>` replacing `discoveryInProgress`. 6 Refs + 1 Deferred-Ref.
- **`suiteToClasses` NOT in service** — writers `resolveSuiteChildren` (1185 `.set`) and `clearTestItems` (879 `.clear()`) stay on shell (4.2 scope); moved methods don't read/write it (verified: reads at 489/528/570/1242/1284/1316 all in shell methods); two-store divergence if moved now. Stays shell class field; migrates to Ref in 4.2 with `resolveSuiteChildren`. `clearTestItems` clears `suiteToClasses` directly and calls service `reset` accessor for the moved Maps.
- `controller` (vscode.TestController) + tags stay on shell (vscode lifecycle); pass to service methods as params (runtime data) — service holds only the Maps/flags Refs.
- **Dedup (Deferred):** `discover` reads the in-flight `Ref<Option<Deferred>>`; if `Some`, `Deferred.await` it and return; if `None`, `Deferred.make`, set Ref to `Some`, run the discovery body, `Deferred.succeed`/`failCause` on completion and reset Ref to `None` in `Effect.ensuring`. Single-shot per `refresh`; not memoized.
- **isRestoringResults:** `Ref.modify` test-and-set (`prev ? [skip, prev] : [proceed, true]`) at entry of `restorePreviousResults`; reset in `ensuring`. Atomic, no class field.
- **Restore-results apply loop (`restorePreviousResults`):** the per-URI `ctx.updateTestResults(uri)` apply must be `Effect.forEach(recentUris, uri => Effect.tryPromise({ try: () => ctx.updateTestResults(uri), catch: e => new RestoreResultsError({ uri: uri.toString(), message: errorToString(e) }) }), { concurrency: 1 })` — NOT a `for...await` loop wrapped in one outer `tryPromise`. SKILL.md: 'forEach/for driving effects → Effect.forEach'. The `catch` lives on each per-item `tryPromise` so a failure carries the offending URI (add `uri: Schema.String` to `RestoreResultsError`), instead of bucketing every per-URI failure into one opaque error. `concurrency: 1` preserves the deliberate oldest-first ordering (most recent result wins per method).
- **yield batching:** `populateTestItemsFromOrg` setImmediate/Promise loop → `Effect.yieldNow()` at the `processed % BATCH_SIZE === 0` boundary.
- `discover`/`refresh`/`populateSuiteItems`/`populateTestItemsFromOrg`/`restorePreviousResults` → `Effect.fn` (span names) on service.
- **Errors (declared Phase 1):**
  - `DiscoveryError` (`Schema.TaggedError`, `{ message: Schema.String }`) — top-level discovery failure surfaced to user.
  - `PackageResolutionError` (`Schema.TaggedError`, `{ message: Schema.String }`) — maps the tooling-query error re-thrown out of `resolvePackage2Members` (Phase 2 wraps that call with `Effect.tryPromise({ try, catch: e => new PackageResolutionError({ message: ... }) })`).
  - `SuiteRetrievalError` (`Schema.TaggedError`) — for `retrieveAllSuites` failure (replaces console.error at 982; today returns early, keep that behavior by `catchTag`→`logError`→empty).
  - `RestoreResultsError` (`Schema.TaggedError`, `{ uri: Schema.String, message: Schema.String }`) — restore path failures (replaces console.debug at 427). `uri` carries the offending result file so a per-item failure (from the `Effect.forEach` apply loop) identifies which URI failed.
  - `@ExportTaggedError` only if any cross the pkg boundary (none consumed cross-pkg today → keep internal unless ts4023 LS flags it).
- **`doDiscover` handler:** `catchTags({ DiscoveryError, PackageResolutionError, SuiteRetrievalError, RestoreResultsError })` → notify (warning when message === `apex_test_discovery_partial_warning`, else error). No `catchAll`, no `console.debug`. Constraint: zero bare `UnknownException` — every `tryPromise` must have typed `catch` mapping to a declared error.
- **Sync snapshot getters:** service exposes `getMethodItemsSnapshot`/`getClassItemsSnapshot`/etc. = `runtime.runSync(Ref.get(ref))` read-only projections, for the sync vscode TestRun callers (`applyStaleTags`/`clearStaleTagsForTests`) that cannot await. These are the only `runSync` uses and are the deliberate vscode-sync bridge.
- Shell `refresh()`/`discoverTests()` become thin `runPromise` wrappers over service effects (public Promise API preserved for consumers).
- Register `ApexTestTreeService.Default` in `extensionProvider.ts` `Layer.mergeAll`.

### Phase 2 — move discovery methods to service

- Move `populateSuiteItems`/`populateTestItemsFromOrg`/`restorePreviousResults`/`doDiscoverTestsEffect` bodies into service `Effect.fn`s, reading/writing Refs (no class fields).
- **Per-`tryPromise` typed catch:** every `Effect.tryPromise` in the moved bodies gets `{ try, catch }` mapping to a declared tagged error — no bare `UnknownException` left to bucket. Specifically:
  - `resolvePackage2Members` call (702/907, inside `populateTestItemsFromOrg`) → `catch: e => new PackageResolutionError({ message: errorToString(e) })`.
  - `retrieveAllSuites` (980) → `SuiteRetrievalError`; preserve current early-return behavior via `catchTag('SuiteRetrievalError', e => Effect.logError(...).pipe(Effect.as([])))`.
  - restore-path apply: the per-URI `ctx.updateTestResults` loop → `Effect.forEach(recentUris, uri => Effect.tryPromise({ try: () => ctx.updateTestResults(uri), catch: e => new RestoreResultsError({ uri: uri.toString(), message: errorToString(e) }) }), { concurrency: 1 })`. NO `for...await` inside a single outer `tryPromise` (current shape at apexTestTreeService.ts:383-390 is wrong — per-item catch, not one bucket). `concurrency: 1` keeps oldest-first ordering, so the most recent result wins per method.
  - the scan-phase promises (stat/read per URI, ~341-369) already use `Effect.forEach`/per-item `tryPromise`; each failing op constructs `RestoreResultsError` with that op's `uri`. Every `RestoreResultsError` carries a real URI — no empty-string placeholder.
- **console.* → Effect.log\*** at the moved sites (each called out in Context): 964 `Effect.logError`, 982 `Effect.logError`, 427 `Effect.logWarning`, 148 `Effect.logWarning` (clearResults). 1140/1168/1543 stay (shell/4.2) — flag, don't migrate.
- `populateTestItemsFromOrg`: replace setImmediate/Promise batching w/ `Effect.yieldNow()`.
- `discover` w/ Deferred dedup (`Ref<Option<Deferred>>` create/await/complete-in-ensuring; replaces `discoveryInProgress` Promise + `doDiscoverTests` wrapper).
- `restorePreviousResults`: `Ref.modify` test-and-set on `isRestoringResults`; reset in `ensuring`.
- **`doDiscover` handler:** `catchTags({ DiscoveryError, PackageResolutionError, SuiteRetrievalError, RestoreResultsError })` → notify (warning vs error per `apex_test_discovery_partial_warning`). Remove the `catchAll`+`console.debug` (244) entirely. No catchAll/swallow.
- Pass `controller` + tags as method params.
