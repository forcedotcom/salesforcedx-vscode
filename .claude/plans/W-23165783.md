## Design decisions

- New `views/apexTestTreeService.ts` (or `services/`): `Effect.Service` `ApexTestTreeService`, `accessors:true`, `dependencies:[]` (reach ambient services via `api.services`).
- Owns Refs: `suiteItems`/`classItems`/`methodItems`/`classToParentItem` (`Ref<Map>`), `hasRestoredResults`/`isRestoringResults` (`Ref<boolean>`), plus the in-flight `Ref<Option<Deferred<void, DiscoveryError>>>` replacing `discoveryInProgress`. **`suiteToClasses` NOT here** — stays a shell field (writer out-of-scope; see Context). 6 Refs + 1 Deferred-Ref.
- `controller` (vscode.TestController) + tags stay on shell (vscode lifecycle); pass to service methods as params (runtime data) — service holds only the Maps/flags Refs.
- **Dedup (Deferred):** `discover` reads the in-flight `Ref<Option<Deferred>>`; if `Some`, `Deferred.await` it and return; if `None`, `Deferred.make`, set Ref to `Some`, run the discovery body, `Deferred.succeed`/`failCause` on completion and reset Ref to `None` in `Effect.ensuring`. Single-shot per `refresh`; not memoized.
- **isRestoringResults:** `Ref.modify` test-and-set (`prev ? [skip, prev] : [proceed, true]`) at entry of `restorePreviousResults`; reset in `ensuring`. Atomic, no class field.
- **yield batching:** `populateTestItemsFromOrg` setImmediate/Promise loop → `Effect.yieldNow()` at the `processed % BATCH_SIZE === 0` boundary.
- `discover`/`refresh`/`populateSuiteItems`/`populateTestItemsFromOrg`/`restorePreviousResults` → `Effect.fn` (span names) on service.
- **Errors (declared Phase 1):**
  - `DiscoveryError` (`Schema.TaggedError`, `{ message: Schema.String }`) — top-level discovery failure surfaced to user.
  - `PackageResolutionError` (`Schema.TaggedError`, `{ message: Schema.String }`) — maps the tooling-query error re-thrown out of `resolvePackage2Members` (Phase 2 wraps that call with `Effect.tryPromise({ try, catch: e => new PackageResolutionError({ message: ... }) })`).
  - `SuiteRetrievalError` (`Schema.TaggedError`) — for `retrieveAllSuites` failure (replaces console.error at 982; today returns early, keep that behavior by `catchTag`→`logError`→empty).
  - `RestoreResultsError` (`Schema.TaggedError`) — restore path failures (replaces console.debug at 427).
  - `@ExportTaggedError` only if any cross the pkg boundary (none consumed cross-pkg today → keep internal unless ts4023 LS flags it).
- **`doDiscover` handler:** `catchTags({ DiscoveryError: ..., PackageResolutionError: ..., SuiteRetrievalError: ..., RestoreResultsError: ... })` → notify (warning when message === `apex_test_discovery_partial_warning`, else error). NO `catchAll`, NO `console.debug`. Any residual `UnknownException` from un-narrowed `Effect.tryPromise` boundaries must be eliminated by giving each `tryPromise` a typed `catch` mapping to one of the above — do not leave a catch-all bucket.
- **Sync snapshot getters:** service exposes `getMethodItemsSnapshot`/`getClassItemsSnapshot`/etc. = `runtime.runSync(Ref.get(ref))` read-only projections, for the sync vscode TestRun callers (`applyStaleTags`/`clearStaleTagsForTests`) that cannot await. These are the only `runSync` uses and are the deliberate vscode-sync bridge.
- Shell `refresh()`/`discoverTests()` become thin `runPromise` wrappers over service effects (public Promise API preserved for consumers).
- Register `ApexTestTreeService.Default` in `extensionProvider.ts` `Layer.mergeAll`.
