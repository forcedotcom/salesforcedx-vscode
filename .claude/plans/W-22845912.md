`helpers.ts` mixes ~100-line pattern arrays with runtime helpers. Extract each array to its own file for readability/maintainability. Pure move — no behavior change.

### Phase 2: Add unit tests for filterErrors / filterNetworkErrors

The existing e2e spec (`helpers.headless.spec.ts`) only exercises `waitForVSCodeWorkbench`, `closeWelcomeTabs`, and `waitForWorkspaceReady` — it does not test `filterErrors` or `filterNetworkErrors`. Since these are pure functions (no Page dependency), add a Playwright test file that directly imports and exercises them with known inputs.

- Create `packages/playwright-vscode-ext/test/playwright/specs/filterErrors.spec.ts`
  - Import `filterErrors` and `filterNetworkErrors` from `../../../src/utils/helpers`
  - Test: a `ConsoleError` whose text matches a known pattern (e.g. `'favicon.ico'`) is filtered out
  - Test: a `ConsoleError` whose text does NOT match any pattern passes through
  - Test: a `NetworkError` whose url matches a known network pattern is filtered out
  - Test: a `NetworkError` whose url does NOT match any pattern passes through
  - Use `@playwright/test` `test`/`expect` (already available; no new deps) — these are logic-only tests that do not need a browser fixture
- Commit: `test: add unit tests for filterErrors/filterNetworkErrors - W-22845912`
