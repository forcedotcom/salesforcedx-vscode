# W-23250969 — migrate `sf.config.set` → `ConfigService.setTargetOrg`

## Context

- Replace `sf.config.set` vscode command w/ `ConfigService.setTargetOrg(usernameOrAlias)` Effect in `salesforcedx-vscode-services`.
- Sole runtime caller: org picker `orgPicker/orgList.ts:209` (`executeCommand('sf.config.set', usernameOrAlias)`). Not in palette. Pure `@salesforce/core` `Config`, no CLI.
- `ConfigService` owns `unsetTargetOrg`/`unsetTargetDevHub`, no setter — fix asymmetry.
- Status-bar refresh **driven by the picker, not the file watcher**. `setTargetOrg` writes config + invalidates aggregator. Picker then calls `invalidateCachedConnections()` + `getConnection()` → background `SubscriptionRef.set(defaultOrgRef, …)` (`connectionService.ts:371`, forked fiber). Identical desktop + web; no dependency on `createFileSystemWatcher`.
- `setTargetOrg` lives in `ConfigService`; it must NOT call `ConnectionService` (would create a ConfigService→ConnectionService cycle — ConnectionService already depends on ConfigService `connectionService.ts:205`). The connection refresh is orchestrated in the picker (`orgList.ts`), which already holds `api.services` for both.
- Picker filters expired orgs (`orgList.ts:162` `.filter(o => o.isExpired !== true)`) → drop old `Org.create` expiry/existence check. `Org.create` in old `setTargetOrgOrAlias` (`orgUtil.ts:146`) was an existence-validation side effect before write; picker only offers known, non-expired orgs so it is redundant. Accept seconds-wide TOCTOU. No `process.chdir` (legacy; core sets cwd at activation, `unsetTargetOrg` writes local config w/o chdir).
- **No precondition guard.** Activation gated by `workspaceContains:sfdx-project.json` (`package.json:197-198`) + `sf.set.default.org` `"when": "sf:project_opened"` (`package.json:264-265`) — precondition structurally guaranteed; dropping old `sfProjectPreconditionChecker` is deliberate.

### Findings (deviations from WI text)

- `updateConfigAndStateAggregators` / internal `refreshConnection` in `orgUtil.ts`: WI says drop "if no other caller" — they HAVE many callers (orgDelete, orgList, orgCreate, all auth commands). KEEP both.
- Removable from `orgUtil.ts`: only `setTargetOrgOrAlias` (export) + `setUsernameOrAlias` (internal) — sole consumer is `configSet.ts`.
- `config_set_org_text` nls key: KEEP — `sf.set.default.org` palette title uses `%config_set_org_text%` (`package.json:320`). Removable nls keys (only `configSet.ts` uses): `config_set_name`, `table_header_name`, `table_header_value`, `table_header_success`.
- e2e `orgPicker.desktop.spec.ts` asserts `SFDX: Set a Default Org successfully ran` toast (4x) + output-channel `target-org` table. Post-migration: no commandlet → no toast, no table. Spec MUST change to status-bar-only assertions. On-branch e2e work, part of this change.
- Two more desktop specs reference `config_set_org_text` and must be handled (Phase 4):
  - `orgCommands.desktop.spec.ts:33` `verifyCommandExists(page, packageNls.config_set_org_text, 60_000)` — checks the palette title `SFDX: Set a Default Org`. `sf.set.default.org` keeps title `%config_set_org_text%` (`package.json:319-320`), so the command still resolves. **No change needed; confirm it still passes** (do NOT delete).
  - `telemetryIdentitySeeding.desktop.spec.ts:72` `executeCommandWithCommandPalette(page, packageNls.config_set_org_text)` — title now maps to `sf.set.default.org`, which opens a modal `showQuickPick` and hangs the spec. Switch to `sf.set.default.org` + `Escape` dismissal, or a non-blocking activation command. See Phase 4.
- No web playwright harness exists in this package — only `playwright.config.desktop.ts` (`test/playwright/`). There is NO web/headless config, memfs fixture, or wireit script for `salesforcedx-vscode-org`. A new `orgPicker.web.spec.ts` would require standing up an entire web harness. The plan instead removes the web-watcher risk at the source (picker-driven ref refresh) so no web e2e is required to de-risk this change — see Adversarial concerns.

### Phase 2 — picker calls service; remove command

- `orgPicker/orgList.ts:208-209`: `usernameOrAlias` is already `selection.orgAlias ?? selection.orgUsername ?? ''` where `orgAlias = orgAuth.aliases?.[0]` (`orgList.ts:121`) — a single alias, never comma-separated. Replace `vscode.commands.executeCommand('sf.config.set', usernameOrAlias)` with:
  - `yield* api.services.ConfigService.setTargetOrg(usernameOrAlias)` (write config; `api` already yielded `:189`).
  - then `yield* api.services.ConnectionService.invalidateCachedConnections()` + `yield* api.services.ConnectionService.getConnection()` to trigger the background `defaultOrgRef` refresh (forked `SubscriptionRef.set` at `connectionService.ts:371`); status bar updates reactively via the ref's `changes` stream, desktop + web, no file watcher. `catchTags` over the typed error set so a bad selection doesn't crash the picker effect.
  - **Do NOT add `.split(',')[0]`.** The old `configSet.ts:32` split operated on a raw `executeCommand` argument; the picker already passes a single `aliases?.[0]`. A no-op split is misleading and could mangle an alias containing a comma. Pass the value unchanged.
- Remove `commands/configSet.ts`.
- `commands/index.ts:8`: drop `export { configSet }`.
- `index.ts`: drop `configSet` import (`:20`) + registration `registerCommand('sf.config.set', configSet)` (`:41`).
- `util/orgUtil.ts`: remove `setTargetOrgOrAlias` (export) + `setUsernameOrAlias` (internal). Drop now-unused imports (`Org`; `OrgConfigProperties`/`Config`/`workspaceUtils` only if no remaining use — verify).
- `messages/i18n.ts` + `messages/i18n.ja.ts`: remove `config_set_name`, `table_header_name`, `table_header_value`, `table_header_success`. KEEP `config_set_org_text`.
