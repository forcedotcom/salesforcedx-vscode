# W-23031858 — Make soql-model a real internal package (ADR + conversion)

## Context

`src/soql-model/` in `salesforcedx-vscode-soql` = quasi-package. Imported as `@salesforce/soql-model/*`; no package.json/exports/boundary. Specifier faked by 4 alias mechanisms (none resolves a real pkg). Tail of soql-tooling→vscode move (PR #6821).

Decision: real INTERNAL workspace pkg `@salesforce/soql-model`, consumed via `*`, NOT published. Invariant: soql-common must never import soql-model (would force publish).

Layout = mirror `packages/soql-common` (sibling internal pkg, `out/` build, wireit compile/lint/test, jest).

Findings (verified):
- Source: `src/soql-model/{analyzers,messages,model,model/impl(15 files),serialization,validators}`.
- Internal imports all relative → survive move unchanged.
- External deps: `@salesforce/soql-common` (soqlComments, soql-parser.lib, generated/*), `antlr4ts`. No vscode/effect/core. Web-safe.
- src consumers: 8 files, 28 imports. soqlUtils.ts holds all 15 `model/impl/*` deep imports (new'd ctors) + `model/util`+`serialization/*`. queryDataHelper.ts: `analyzers/selectAnalyzer`. 6 builder-ui files: `model/model`, `validators/*`.
- Distinct specifiers map to 6 subpaths: `model/model`, `model/util`, `model/impl/*`(15), `serialization/{serializer,deserializer}`, `validators/{validatorFactory,inputUtils}`, `analyzers/selectAnalyzer`.
- Curated exports (WI): `./model` (barrel = model.ts types/enums + 15 impl classes — collapses deep impl imports), `./model/util`, `./analyzers`, `./serialization` (serializer+deserializer), `./validators` (validatorFactory+inputUtils). `messages/i18n` internal-only — not exported.
- 4 fakes: soql `tsconfig.json` paths alias; soql `jest.config.js` moduleNameMapper; `soql-builder-ui/jest.config.js` mapper; `soql-builder-ui/rollup.config.mjs` `@rollup/plugin-alias` entry + comments re aliased sibling.
- esbuild: no tsconfig-paths reliance (grep clean); desktop entry `out/src/index.js` (already-compiled). Web `web.mjs` resolveExtensions only.
- Tests: 36 files under `test/jest/soql-model/**`, all deep-relative (`../../../../src/soql-model/...`), zero alias. NO files at `test/jest/` root (`ls` clean). The "dupes" are 3 FLAT files inside `test/jest/soql-model/` (`serializer.test.ts`, `deserializer.test.ts`, `selectAnalyzer.test.ts`) alongside nested copies (`serialization/serializer.test.ts`, `serialization/deserializer.test.ts`, `analyzers/selectAnalyzer.test.ts`):
  - `serializer` + `selectAnalyzer`: flat vs nested IDENTICAL (excluding imports) → flat is stale dupe, drop flat (2 files).
  - `deserializer`: NOT identical. Flat (782 lines) uses full `kind`-discriminated shapes + `toEqual` (stronger). Nested (721 lines) uses stripped shapes + `toMatchObject` (weaker). Both touched by same recent eslint commit — divergent tests, not stale. Keep the stronger flat version; drop the weaker nested one (drop 1 file). Net: 36 → 33 unique tests, no coverage lost.
- Next ADR num: 0019.

Up-stream-only: `vscode-soql → soql-model → soql-common`. No cycle.

### Phase 2-5 — Create pkg, move source, rewire consumers, tear down fakes, wireit (ONE atomic commit)
NOTE: phases 2-5 ship as a SINGLE commit. Splitting them produces non-compiling intermediate commits (the pre-commit hook compiles+lints and rejects them): moving source without rewiring consumers breaks resolution, and removing aliases before the real pkg resolves breaks the build. They are logically distinct steps but one shippable unit.
- commit (combined): `feat(soql-model): create internal package, move source, rewire consumers and build - W-23031858`

#### Step 2 — Create pkg + move source
- `git mv packages/salesforcedx-vscode-soql/src/soql-model packages/soql-model/src` (preserve history; relative internal imports unchanged).
- Add `packages/soql-model/package.json`: name `@salesforce/soql-model`, `private: true`, no version/publishConfig. deps `@salesforce/soql-common`:`*`, `antlr4ts`:`^0.5.0-alpha.3`. `main`/`types` → `out/...`. `exports` map (collapse impl behind `./model`):
  - `./model` → barrel `out/model/index.{js,d.ts}`
  - `./model/util` → `out/model/util.{js,d.ts}` (single file, no barrel — re-pointing a barrel at one file adds nothing)
  - `./analyzers`, `./serialization`, `./validators` → respective barrels
- Add `packages/soql-model/src/model/index.ts` barrel: re-export `./model` + 15 `./impl/*`. The `model.ts` types/enums and impl classes are distinct symbols — no name collision.
- `export type` directive does NOT apply to this barrel. Reasons:
  - The barrel is exported via `./model` → `out/model/index.js` (tsc-COMPILED), not raw source. rollup/Babel never parse it, so Babel's syntactic erasure (which needs `export type`) never touches it. Babel only sees soql-builder-ui's own `.ts` source (the `toolingModelService.ts` `export type { ToolingModelJson }` fix referenced in rollup.config.mjs comment lines 118-122 was a source-file fix, not a barrel fix).
  - Even if it were Babel-parsed, blanket `export type` would be WRONG here: `model.ts` mixes runtime + types (exports `SyntaxOptions` class AND types/enums via `export * from './model'` → `./unmodeled` which exports `const REASON_*` values + a type). `export type` on a module that exports runtime values would erase those values and break the build.
  - Correct barrel form: `export *` for every module that exports any runtime symbol (model.ts via unmodeled has consts + SyntaxOptions class; all 15 impl files export classes). `export type` ONLY on a re-export that is provably 100% type-only. Audit each re-exported module; default to `export *`.
- Barrel export contents (explicit):
  - `analyzers/index.ts` → `export * from './selectAnalyzer'` (only consumed symbol: `selectAnalyzer`).
  - `serialization/index.ts` → re-export `./serializer` + `./deserializer`.
  - `validators/index.ts` → re-export `./validatorFactory` + `./inputUtils`.
- `model/util.ts` exports exactly one symbol `SoqlModelUtils` (const); consumed by builder-ui `soqlUtils.ts` and internally by `serialization/deserializer.ts`. No barrel — keep direct file in exports map above.
- Add `packages/soql-model/tsconfig.json` (extends `../../tsconfig.common.json`, rootDir `.`, outDir `out`, include `src/**/*.ts`).
- Root `package.json` workspaces auto-include `packages/*` — confirm glob present. RUN `npm install` (not just "confirm") so npm creates the workspace symlink. VERIFY: `ls -la node_modules/@salesforce/soql-model` resolves to `../../packages/soql-model` (symlink). wireit + jest moduleNameMapper-free resolution both depend on this symlink existing.
