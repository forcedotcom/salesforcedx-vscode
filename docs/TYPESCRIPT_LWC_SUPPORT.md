# TypeScript Support for Lightning Web Components

The Salesforce Extensions for VS Code now fully support creating and developing Lightning Web Components (LWC) using TypeScript.

## Quick Start

### Creating a TypeScript Project

Use the Salesforce CLI to create a new TypeScript project:

```bash
sf project generate --name my-typescript-project --lwc-language typescript
```

This creates a project with:

- `sfdx-project.json` with `"defaultLwcLanguage": "typescript"`
- `tsconfig.json` configured for LWC development
- TypeScript dependencies in `package.json`
- VS Code settings pre-configured for TypeScript support

### Creating TypeScript Components

Once in a TypeScript project, the extension automatically creates TypeScript components:

**Command Palette:**

1. Open Command Palette (`Cmd/Ctrl + Shift + P`)
2. Type "SFDX: Create Lightning Web Component"
3. Enter component name
4. TypeScript will be used automatically (if `defaultLwcLanguage: "typescript"`)

**Context Menu:**

1. Right-click on the `lwc` folder in VS Code Explorer
2. Select "SFDX: Create Lightning Web Component"
3. Enter component name
4. TypeScript will be used automatically

### Manual Language Selection

If your project doesn't have `defaultLwcLanguage` configured, JavaScript is used by default.
Set `defaultLwcLanguage` to `"typescript"` to generate TypeScript components by default.

## Project Configuration

### sfdx-project.json

Set your preferred LWC language in `sfdx-project.json`:

```json
{
  "packageDirectories": [
    {
      "path": "force-app",
      "default": true
    }
  ],
  "defaultLwcLanguage": "typescript",
  "sourceApiVersion": "62.0"
}
```

**Supported values:**

- `"typescript"` - Generate TypeScript components by default
- `"javascript"` - Generate JavaScript components by default
- Field not present - Default to JavaScript

### TypeScript Configuration

TypeScript projects include a `tsconfig.json` generated by `@salesforce/templates` with LWC-optimized settings. Key LWC-specific compiler options include:

```jsonc
{
  "extends": ".sfdx/tsconfig.sfdx.json",
  "compilerOptions": {
    "erasableSyntaxOnly": true, // Required for Salesforce type stripping
    "experimentalDecorators": false, // LWC uses standard decorators
    "target": "ESNext",
    "module": "ESNext",
    "moduleResolution": "bundler"
    // ... additional TypeScript settings
  }
}
```

**Important Settings:**

- `erasableSyntaxOnly: true` - Enables type stripping for LWC deployment
- `experimentalDecorators: false` - LWC uses TC39 standard decorators, not legacy experimental decorators
- `extends: ".sfdx/tsconfig.sfdx.json"` - Inherits Salesforce-specific base configuration

> For the complete `tsconfig.json` configuration, refer to your generated project or the [@salesforce/templates](https://github.com/forcedotcom/salesforcedx-templates) repository.

## Component Structure

### TypeScript Component Example

```typescript
// helloWorld.ts
import { LightningElement, api } from 'lwc';

export default class HelloWorld extends LightningElement {
  @api greeting: string = 'World';

  get message(): string {
    return `Hello, ${this.greeting}!`;
  }

  handleClick(event: MouseEvent): void {
    console.log('Button clicked');
  }
}
```

### TypeScript Test Example

```typescript
// __tests__/helloWorld.test.ts
import { createElement } from 'lwc';
import HelloWorld from 'c/helloWorld';

describe('c-hello-world', () => {
  afterEach(() => {
    while (document.body.firstChild) {
      document.body.removeChild(document.body.firstChild);
    }
  });

  it('displays greeting', () => {
    // Type-safe component creation
    const element: HelloWorld = createElement('c-hello-world', {
      is: HelloWorld
    }) as HelloWorld;

    // TypeScript ensures greeting is a string
    element.greeting = 'TypeScript';
    document.body.appendChild(element);

    // Type-safe DOM querying with HTMLElement
    const message: HTMLElement | null = element.shadowRoot?.querySelector<HTMLElement>('.greeting') ?? null;
    expect(message?.textContent).toBe('Hello, TypeScript!');
  });

  it('handles click event with proper typing', () => {
    const element: HelloWorld = createElement('c-hello-world', {
      is: HelloWorld
    }) as HelloWorld;
    document.body.appendChild(element);

    // Type-safe button reference
    const button: HTMLButtonElement | null = element.shadowRoot?.querySelector<HTMLButtonElement>('button') ?? null;

    // Properly typed MouseEvent
    const mockEvent: MouseEvent = new MouseEvent('click');
    button?.dispatchEvent(mockEvent);

    expect(button).not.toBeNull();
  });
});
```

## VS Code Features

### IntelliSense

TypeScript components get full IntelliSense support:

- Auto-completion for LWC decorators (`@api`, `@track`, `@wire`)
- Type checking for properties and methods
- Parameter hints for LWC APIs
- Inline error highlighting

### Go to Definition

Navigate to LWC type definitions:

1. `Cmd/Ctrl + Click` on any LWC import
2. VS Code opens the type definition file
3. Works for all `lwc` module imports

### Type Checking

The extension performs live type checking as you type:

- Underlines type errors in red
- Hover over errors for detailed messages
- Quick fixes available for common issues

## Building & Compiling

### Local Development

Compile TypeScript for local testing:

```bash
npm run build        # Compile once
npm run build:watch  # Watch mode
```

Compiled JavaScript appears in the `dist/` folder for local validation.

### Deployment

**Important:** Salesforce server performs type stripping automatically. You deploy **raw `.ts` files**, not compiled JavaScript.

```bash
sf project deploy start --source-dir force-app
```

The `.forceignore` automatically excludes `dist/` from deployment.

## Migration Guide

### From JavaScript to TypeScript

1. **Update project configuration:**

   ```json
   {
     "defaultLwcLanguage": "typescript"
   }
   ```

2. **Add TypeScript configuration:**
   - Copy `tsconfig.json` from a new TypeScript project template
   - Run `npm install typescript @types/node`

3. **Rename files:**
   - `.js` → `.ts`
   - `.js-meta.xml` stays the same

4. **Add type annotations:**

   ```typescript
   // Before (JavaScript)
   export default class MyComponent extends LightningElement {
     value = '';
     handleChange(event) {
       this.value = event.target.value;
     }
   }

   // After (TypeScript)
   export default class MyComponent extends LightningElement {
     value: string = '';
     handleChange(event: Event): void {
       this.value = (event.target as HTMLInputElement).value;
     }
   }
   ```

### From Hidden Preview Flag

**Old way (deprecated):**

```jsonc
// .vscode/settings.json
{
  "salesforcedx-vscode-lwc.preview.typeScriptSupport": true
}
```

**New way:**

```jsonc
// sfdx-project.json
{
  "defaultLwcLanguage": "typescript"
}
```

The `preview.typeScriptSupport` flag is no longer needed. TypeScript is now a first-class, fully supported option.

## Troubleshooting

### TypeScript Not Available in Component Creation

**Issue:** Only JavaScript is shown when creating components

**Solution:** Check your `sfdx-project.json`:

```json
{
  "defaultLwcLanguage": "typescript"
}
```

If the field is missing, JavaScript is used by default.

### IntelliSense Not Working

**Issue:** No auto-completion or type checking

**Solutions:**

1. Ensure `tsconfig.json` exists in project root
2. Reload VS Code window: `Cmd/Ctrl + Shift + P` → "Developer: Reload Window"
3. Check TypeScript version: `npx tsc --version` (should be 5.0+)

### Build Errors

**Issue:** `tsc` command not found

**Solution:**

```bash
npm install typescript --save-dev
```

### Deploy Errors

**Issue:** Server rejects `.ts` files

**Solution:** Ensure your org supports TypeScript:

- API version 62.0 or higher
- Contact Salesforce support if issues persist
