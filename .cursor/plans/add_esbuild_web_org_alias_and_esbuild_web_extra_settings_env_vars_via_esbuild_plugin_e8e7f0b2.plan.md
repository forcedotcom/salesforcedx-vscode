# Add ESBUILD_WEB_ORG_ALIAS and ESBUILD_WEB_EXTRA_SETTINGS env vars via esbuild plugin

## Overview

Create a separate wireit script that runs `sf org display` (always runs, no caching) and an esbuild plugin that transforms the `vscodeSettings` module during web bundling to inject:

1. Org credentials from `.org-display-output.json` file (generated by wireit script)
2. Additional settings from `.esbuild-web-extra-settings.json` at repo root (if file exists)

The plugin only runs for the web build and doesn't modify any source files.

## Implementation Details

### 1. Create separate wireit script for org display

**File**: `packages/salesforcedx-vscode-services/package.json`

Create a new wireit script `prepare:web-settings`:

- Command: Node script that reads `ESBUILD_WEB_ORG_ALIAS` env var
- If set, runs `sf org display -o ${alias} --json` and writes output to `.org-display-output.json`
- If not set, writes empty object `{}` to `.org-display-output.json`
- No caching: set `files: []` and `output: []` OR don't set them (script always runs)
- Mark `ESBUILD_WEB_ORG_ALIAS` as external env var

**Script location**: Create `packages/salesforcedx-vscode-services/scripts/prepareWebSettings.mjs`

### 2. Create esbuild plugin in esbuild.config.mjs

**File**: `packages/salesforcedx-vscode-services/esbuild.config.mjs`

Create a plugin function that:

- Uses `onLoad` to intercept `vscodeSettings.ts` module loading
- Reads `.org-display-output.json` file (from package directory)
  - Parses JSON to extract `instanceUrl`, `accessToken`, and `version` (API version)
  - Generates settings lines for the three config keys
- Reads `.esbuild-web-extra-settings.json` from repo root (if file exists)
  - Parses and merges its contents
- Transforms the module source code to inject the new settings into the array
- Returns transformed contents with `loader: 'ts'`

**Plugin structure**:

```javascript
const vscodeSettingsTransformPlugin = () => ({
  name: 'vscode-settings-transform',
  setup(build) {
    build.onLoad({ filter: /vscodeSettings\.ts$/ }, async args => {
      // Read original file
      const originalContents = await fs.promises.readFile(args.path, 'utf8');

      // Parse current settings array
      // Extract org settings if ESBUILD_WEB_ORG_ALIAS is set
      // Read and parse extra settings if ESBUILD_WEB_EXTRA_SETTINGS is set
      // Transform the array to include new settings
      // Return transformed contents
    });
  }
});
```

### 2. Apply plugin only to web build

**File**: `packages/salesforcedx-vscode-services/esbuild.config.mjs`

- Add the plugin to `commonConfigBrowser.plugins` array (spread existing plugins)
- Do NOT add it to `nodeConfig` (desktop build)

### 3. Handle wireit caching

**File**: `packages/salesforcedx-vscode-services/package.json`

Update `vscode:bundle` wireit config:

- Add `env` section with both env vars marked as `external: true`
- For `ESBUILD_WEB_ORG_ALIAS`: Write org display output to `.org-display-output.json` in the plugin
- Add `.org-display-output.json` to `files` array (if it exists)
- For `ESBUILD_WEB_EXTRA_SETTINGS`: Add the file path to `files` array when env var is set (but wireit can't do conditional files)

**Better approach**:

- Always write `.org-display-output.json` if `ESBUILD_WEB_ORG_ALIAS` is set (even if empty)
- Add `.org-display-output.json` to `files` array unconditionally
- For `ESBUILD_WEB_EXTRA_SETTINGS`, document that users should add the file path to `files` manually, OR we can add it programmatically if the env var is set

**Wireit config**:

```json
{
  "vscode:bundle": {
    "command": "node esbuild.config.mjs",
    "dependencies": ["compile"],
    "env": {
      "ESBUILD_WEB_ORG_ALIAS": {
        "external": true
      },
      "ESBUILD_WEB_EXTRA_SETTINGS": {
        "external": true
      }
    },
    "files": [
      "esbuild.config.mjs",
      "../../scripts/bundling/node.mjs",
      "../../scripts/bundling/web.mjs",
      "out/**",
      ".org-display-output.json"
    ],
    "output": ["dist"]
  }
}
```

### 5. Transform logic details

The plugin needs to:

1. Parse the current `vscodeSettings` array from the TypeScript source
2. Extract existing settings (between `{` and `}`)
3. Add org settings before the closing brace:

- `"salesforcedx-vscode-code-builder-web.instanceUrl": "<value>",`
- `"salesforcedx-vscode-code-builder-web.accessToken": "<value>",`
- `"salesforcedx-vscode-code-builder-web.apiVersion": "<value>",`

1. Add extra settings from JSON file (merge object entries)
2. Handle JSON escaping properly (quotes, newlines, etc.)
3. Reconstruct the array with proper formatting

**Parsing approach**:

- Use regex to extract the array content: `export const vscodeSettings = (\[.*?\]);`
- Parse the array to get current lines
- Insert new settings before the closing `}` line
- Reconstruct the TypeScript export

### 6. Error handling

**prepare:web-settings script**:

- If `ESBUILD_WEB_ORG_ALIAS` is not set: write `{}` to `.org-display-output.json`
- If `sf org display` fails: log error, write `{}` to `.org-display-output.json`

**esbuild plugin**:

- If `.org-display-output.json` doesn't exist or is empty/invalid: skip org settings
- If `.esbuild-web-extra-settings.json` doesn't exist: skip extra settings (no error)
- If extra settings file exists but is invalid JSON: log warning, skip extra settings

### 6. File modifications

**Files to modify**:

1. `packages/salesforcedx-vscode-services/esbuild.config.mjs` - Add plugin creation and apply to web build
2. `packages/salesforcedx-vscode-services/package.json` - Update wireit `vscode:bundle` config
3. `.gitignore` (if needed) - Add `.org-display-output.json`

## Key Points

- Separate wireit script `prepare:web-settings` always runs (no caching) to generate `.org-display-output.json`
- Bundle depends on prepare script and uses output file as input for proper caching
- Extra settings file standardized to `.esbuild-web-extra-settings.json` at repo root (gitignored)
- Plugin only transforms during web build (browser build)
- Source files remain untouched
- Compiled files in `out/` remain untouched
- Transformation happens in-memory during esbuild bundling
- Wireit caching properly invalidates when org display output or extra settings file changes
